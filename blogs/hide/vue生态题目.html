Vue生态面试题 ​

1. Vue2 不能监听数组下标的原因 ​

这是 Vue2 响应式系统的一个重要限制。主要原因在于 Object.defineProperty 的实现机制:

  1. 技术限制
  • Object.defineProperty 只能劫持对象的属性
  • 它不能监听数组的长度变化
  • 不能监听通过索引设置数组元素
  1. 性能考虑
  • 如果监听所有数组索引,会产生大量的 getter/setter
  • 数组长度不固定,无法提前定义索引属性
  • 频繁创建响应式属性会影响性能
  1. Vue2 的解决方案
js
// 1. 使用 Vue.set
Vue.set(array, index, value)

// 2. 使用数组变异方法
array.splice(index, 1, value)

// 3. 替换整个数组
array = [...array.slice(0, index), value, ...array.slice(index + 1)]
  1. 实践建议
  • 优先使用数组变异方法
  • 需要响应式时使用 Vue.set
  • 考虑数据结构设计,避免直接操作索引

5. Vue 的响应式原理 ​

深入剖析 Vue 的响应式系统实现:

  1. 依赖收集
js
// Vue3 响应式核心
function reactive(target) {
  return new Proxy(target, {
    get(target, key) {
      track(target, key) // 收集依赖
      return target[key]
    },
    set(target, key, value) {
      target[key] = value
      trigger(target, key) // 触发更新
      return true
    }
  })
}
  1. 计算属性实现
js
function computed(getter) {
  let value
  let dirty = true
  
  const effect = reactive(() => {
    dirty = true
    value = getter()
  })
  
  return {
    get value() {
      if (dirty) {
        effect()
        dirty = false
      }
      return value
    }
  }
}
  1. 监听器原理
js
// Vue3 watch 实现
function watch(source, callback) {
  let cleanup
  const effect = reactive(() => {
    cleanup?.()
    cleanup = callback(source)
  })
  
  return () => {
    cleanup?.()
    effect.stop()
  }
}

  // 1. 类型不同，直接替换
  if (n1.type !== n2.type) {

    return
  }

16. Vue 的渲染流程 ​

Vue 从模板到最终渲染的完整流程:

  1. 编译阶段
  • 模板解析: 将模板字符串转换为 AST(抽象语法树)
  • 优化阶段: 标记静态节点,提升不变的内容
  • 代码生成: 将 AST 转换为渲染函数
  1. 响应式阶段
  • 数据劫持: 使用 Proxy/defineProperty 监听数据变化
  • 依赖收集: 建立数据与副作用的对应关系
  • 派发更新: 数据变化时触发相关副作用
  1. 虚拟 DOM 阶段
  • 创建 VNode: 执行渲染函数生成虚拟节点树
  • Diff 算法: 对比新旧虚拟节点树找出变化
  • 补丁操作: 将变化同步到真实 DOM

  • 多次数据变化会被缓存在队列中
  • 使用 Promise.then 或 MutationObserver 等微任务执行更新
  • 确保所有数据变化在一个事件循环周期内处理完成
  1. 更新优先级
  • 组件更新按照父到子的顺序
  • watcher 更新按照创建顺序执行
  • 用户的 watcher 优先于渲染 watcher
  1. 手动控制时机
  • nextTick 等待所有更新完成
  • flush: 'post' 在更新后执行回调
  • flush: 'pre' 在更新前执行回调

核心原理示例:

js
const queue = []
let pending = false

function queueJob(job) {
  if (!queue.includes(job)) {
    queue.push(job)
    if (!pending) {
      pending = true
      Promise.resolve().then(flushJobs)
    }
  }
}

// 响应式系统核心
function reactive(target) {
  // 防止重复代理
  if (isProxy(target)) {
    return target
  }
  
  // 创建响应式代理
  return new Proxy(target, {
    get(target, key, receiver) {
      track(target, key)
      return Reflect.get(target, key, receiver)
    },
    set(target, key, value, receiver) {
      const oldValue = target[key]
      const result = Reflect.set(target, key, value, receiver)
      if (hasChanged(value, oldValue)) {
        trigger(target, key)
      }
      return result
    }
  })
}

  constructor() {
    this.events = new Map()
  }
  
  on(event, callback) {
    if (!this.events.has(event)) {
      this.events.set(event, new Set())
    }
    this.events.get(event).add(callback)
  }

      this.events.get(event).forEach(cb => cb(...args))
    }
  }

// 简化的响应式系统
class Dep {
  constructor() {
    this.subscribers = new Set()
  }

  depend() {
    if (activeEffect) {
      this.subscribers.add(activeEffect)
    }
  }

  notify() {
    this.subscribers.forEach(effect => effect())
  }
}

function reactive(obj) {
  return new Proxy(obj, {
    get(target, key) {
      deps[key].depend()
      return target[key]
    },
    set(target, key, value) {
      target[key] = value
      deps[key].notify()
      return true
    }
  })
}
