前端性能题目 ​

1. 性能优化相关的参考指标有哪些? ​

主要包括以下几个关键指标：

  1. FP(First Paint): 首次绘制时间
  • 浏览器首次绘制像素到屏幕上的时间点
  • 通常是页面的第一个非白色像素点
  • 用于评估页面渲染速度
  1. FCP(First Contentful Paint): 首次内容绘制时间
  • 页面首次绘制任何文本、图片、非白色背景等内容的时间点
  • 反映页面内容开始呈现的时间
  • 是重要的用户体验指标
  1. FMP(First Meaningful Paint): 首次有意义绘制时间
  • 页面首次呈现主要有意义内容的时间点
  • 反映用户认为页面有用内容的可见时间
  • 对用户体验有重要影响
  1. TTI(Time to Interactive): 可交互时间
  • 页面变得完全可交互的时间点
  • 用户可以与页面元素进行交互
  • 较短的TTI有助于提供良好的用户体验
  1. TTFB(Time to First Byte): 首字节时间
  • 从发出请求到收到第一个字节响应的时间
  • 反映服务器响应速度
  • 影响整体页面加载速度
  1. DCL(DOM Content Loaded): DOM加载完成时间
  • 浏览器完成解析DOM树的时间点
  • 用于评估页面脚本加载性能
  1. CLS(Cumulative Layout Shift): 累积布局偏移
  • 衡量页面元素布局稳定性
  • 较低的CLS表示页面布局更稳定
  • 直接影响用户体验
  1. LCP(Largest Contentful Paint): 最大内容绘制时间
  • 页面最大内容块(图片/文本)绘制完成的时间
  • 评估页面主要内容加载速度的重要指标
  1. 其他重要指标
  • CPU和内存使用率:监测应用资源占用
  • 网络请求:请求次数和大小优化
  • 渲染性能:帧率(FPS)和渲染时间
  • 页面加载时间:完整加载所需总时间
  • 缓存命中率:评估缓存使用效率

4. 如何实现长缓存 ​

长缓存(Long-term caching)是一种前端性能优化策略，它旨在使浏览器能够缓存应用程序的静态资源(如JavaScript、Css、图像等)更长的时间，以减少不必要的网络请求，加速页面加载速度，并减少带宽消耗。通过将资源文件的内容与它们的文件名关联，可以实现长缓存。通常，浏览器会根据资源文件的URL来判断是否从缓存中获取资源，因此如果资源文件的URL不变，浏览器就会继续使用缓存的资源，直到URL发生变化。

  1. 文件名使用内容哈希 Webpack可以生成包含哈希值的文件名，以确保每个文件在内容发生变化时具有不同的文件名。这样，当文件内容变化时，浏览器会重新下载该文件。
js
    module.exports = {
        output: {
            filename: '[name].[contenthash].js'
        }
    }
  1. 输出文件分离: 将应用程序的代码与第三方库、样式表和其他资源文件分开打包成多个文件。这样，只有在发生变化时才需要重新下载应用程序代码，而其他资源可以长期缓存。
js
    optimization: {
        runtimeChunk: 'single', // 将运行时代码分离到单独的文件中
        splitChunks: {
          cacheGroups: {
            vendor: {
              test: /[\\/]node_modules[\\/]/,
              name: 'vendor',
              chunks: 'all',
            }
          }
        }
    }
  1. 合理的缓存策略 配置缓存控制:在服务器端配置HTTP响应头，以设置资源文件的缓存控制策略，包括 Cache-Control 和Expires 等。这些头部指示浏览器缓存资源的时间。
config
# Nginx 配置示例

location / static / {
    expires 1y;
    add_header Cache- Control "public, no-transform";
}
  1. 版本号控制
  • 在构建时为静态资源添加版本号
  • 可以通过环境变量或git commit hash生成
js
const version = process.env.VERSION || require('child_process')

  • Chunkhash: 基于入口文件及其依赖的hash

  1. 分片处理
  • 使用 requestAnimationFrame 分片处理
  • 设置合理的切片大小
js
function chunk(array, process, context) {
  let items = array.slice(0);
  const CHUNK_SIZE = 1000;

  function doChunk() {
    let chunk = items.splice(0, CHUNK_SIZE);
    chunk.forEach(item => {
      process.call(context, item);
    });
    
    if (items.length > 0) {
      requestAnimationFrame(doChunk);
    }
  }

  requestAnimationFrame(doChunk);
}
  1. Web Worker 并行处理
  • 将数组分割成多个子数组
  • 创建多个 Worker 并行处理
  • 合并处理结果
js
function parallelProcess(array) {
  const WORKER_COUNT = navigator.hardwareConcurrency || 4;
  const chunk = Math.ceil(array.length / WORKER_COUNT);
  const workers = [];

  for(let i = 0; i < WORKER_COUNT; i++) {
    const worker = new Worker('worker.js');
    const start = i * chunk;
    const end = start + chunk;
    
    worker.postMessage({
      array: array.slice(start, end),
      start,
      end
    });
    
    workers.push(worker);
  }
  
  return Promise.all(workers.map(worker => {
    return new Promise(resolve => {
      worker.onmessage = e => resolve(e.data);
    });
  }));
}
  1. 使用优化的循环方式
  • 使用 while 替代 for...of
  • 缓存数组长度
  • 避免数组方法的链式调用
js
// 不推荐
for(const item of array) {
  // 处理
}

// 推荐
let i = 0;
const len = array.length;

6. webWorker 优化 100000000 数组遍历 ​

Web Worker 可以在后台线程中运行计算密集型任务,避免阻塞主线程:

js

    array: new Array(100000000).fill(1)

    console.log('计算结果:', e.data);
};
// worker.js
self.onmessage = function (e) {
    const array = e.data.array;
    const result = array.reduce((acc, cur) => acc + cur, 0);
    self.postMessage(result);
};

7. 延迟加载的方式有哪些? ​

延迟加载(Lazy Loading)是一种用于优化网页性能的策略，可以推迟加载页面资源，以减少初始页面加载时间。

  1. 图片延迟加载 通过将图片的实际地址存储在 data-src 属性中，在适当时机再加载：
js
// 使用 Intersection Observer 实现图片懒加载
const observer = new IntersectionObserver((entries) => {
  entries.forEach(entry => {
    if (entry.isIntersecting) {
      const img = entry.target;
      img.src = img.dataset.src;
      observer.unobserve(img);
    }
  });
});

document.querySelectorAll('img[data-src]').forEach(img => observer.observe(img));
  1. JavaScript 懒加载 使用 async/defer 或动态导入实现脚本延迟加载：

  • defer 属性表示脚本将在页面解析完毕后，DOMContentLoaded 事件触发前执行，但不会阻塞页面渲染

8. 图片懒加载和预加载的区别 ​

懒加载(Lazy Loading):

  • 延迟加载非首屏图片
  • 当图片进入可视区域时才加载
  • 减少首屏加载时间
  • 适用于长列表、图片画廊等

预加载(Preloading):

  • 提前加载后续可能需要的图片
  • 当用户访问时可立即显示
  • 增加首次加载时间
  • 适用于轮播图、相册预览等

function compressImage(file) {
  const canvas = document.createElement('canvas');
  const ctx = canvas.getContext('2d');

  return new Promise((resolve) => {
    img.onload = () => {

  });
}

11. 浏览器的渲染过程，DOM树和渲染树的区别 ​

渲染过程:

  1. 解析HTML构建DOM树
  2. 解析CSS构建CSSOM树
  3. 将DOM和CSSOM合并成渲染树
  4. 布局(Layout/Reflow)
  5. 绘制(Paint)

DOM树和渲染树区别:

  • DOM树包含所有HTML元素
  • 渲染树只包含可见元素
  • display:none的元素在渲染树中不存在
  • visibility:hidden的元素在渲染树中存在

14. 虚拟列表，如果子元素高度不固定，处理方案 ​

  1. 预估高度 + 动态调整
javascript
class DynamicVirtualList {
  constructor(options) {
    this.estimatedItemHeight = options.estimatedItemHeight;
    this.heightCache = new Map(); // 缓存实际高度
    
    this.observer = new ResizeObserver(entries => {
      entries.forEach(entry => {
        const index = entry.target.dataset.index;
        this.heightCache.set(index, entry.contentRect.height);
        this.updatePositions();
      });
    });
  }

  getItemPosition(index) {
    let position = 0;
    for (let i = 0; i < index; i++) {
      position += this.heightCache.get(i) || this.estimatedItemHeight;
    }
    return position;
  }

  updatePositions() {
    // 更新每个元素的位置
    this.items.forEach((item, index) => {
      item.style.transform = `translateY(${this.getItemPosition(index)}px)`;
    });
  }
}

  • 使用多个域名可以增加并发数
