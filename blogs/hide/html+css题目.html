1. 什么是重绘，什么是回流？如何减少回流？ ​

  • 重绘（Re-paint）： 指页面元素在样式改变后，在不影响其布局的情况下，浏览器重新绘制该元素。如字体颜色，背景颜色等。
  • 回流（Reflow）****： 指页面元素在样式改变后，需要重新计算其位置、尺寸等信息，浏览器重新计算该元素。如宽度、高度、位置等。
  • 回流的成本比重绘高，因为回流需要重新计算页面元素的位置、尺寸等信息，而重绘只需要重新绘制页面元素。
  • 如何减少回流：
    • 使用CSS动画代替JS动画： CSS动画利用GPU硬件加速，可以减少浏览器的计算量，提高性能。使用opacity和transform等属性进行动画，而不是改变left、top等属性。
    • 使用transldate3d或者will-change属性可以开启硬件加速：合理使用translate3d(0,0,0)或者will-change属性，可以减少浏览器的计算量，提高性能。
    • **使用requestAnimationFrame：**通过requestAnimationFrame方法，可以确保动画在浏览器的重绘周期内执行，从而避免不必要的回流，这种方式可以确保动画在最佳时间点渲染。
    • **使用文档片段(DocumentFragment)：**当需要在页面中插入大量元素时，可以将元素放入一个文档片段中，然后再一次性插入到页面中，可以减少回流次数。（）
    • 让元素脱离文档流：position:absolute;/position:fixed;/display:none;/visibility:hidden;等属性都可以将元素脱离文档流，从而减少回流(不是避免回流)。
    • 使用visibility:hidden;代替display:none;：visibility:hidden;不会影响页面布局，只是不可见，而display:none;会将元素从渲染树中移除，引起回流。`

2. Margin 塌陷问题如何解决？BFC是什么？怎么触发？ ​

html
<html>
  <head>
    <style>
      .box {
        width: 200px;
        height: 200px;
        background-color: red;
      }
      #box1{
        margin-bottom: 200px;
      }
      #box2{
        margin-top: 100px;
      }
    </style>
  </head>
  <body>
    <div class="box" id="box1"></div>
    <div class="box" id="box2"></div>
  </body>    
</html>
  • **Margin 塌陷问题：**上面例子中，div的间隔是200px，取 margin 重叠部分的更大值（这是正常情况，符合CSS的外边距合并规则），如果希望间距是 300px，需要用到BFC。
  • **BFC：**BFC（Block Formatting Context）是块级格式化上下文，是页面布局中一个重要的概念。BFC是一个独立的容器，容器里面的元素不会影响容器外部的元素，反之亦然。
  • 怎么触发？
    • **float不为none：**float不为none，则会创建BFC。
    • **overflow不为visible：**overflow不为visible，则会创建BFC。
    • **display为inline-block、table-cell、table-caption：**display为inline-block、table-cell、table-caption，则会创建BFC。
    • **position为absolute或fixed：**position为absolute或fixed，则会创建BFC。

4. overflow 有什么值？ ​

属性值效果
visible默认值，内容溢出容器的时候不会被裁剪，会显示在容器之外。意味着溢出的内容会覆盖其他元素
hidden内容溢出容器时会被裁剪，不会显示在容器之外。
scroll内容超出元素框会显示滚动条。
auto根据内容情况，如果内容超出元素框，则会显示滚动条，否则不显示。
inherit从父元素继承overflow属性值。

三栏布局是一种常见的页面布局方式，它通常包含一个自适应的主内容区域、一个左侧侧边栏和右侧侧边栏。

html

        display: flex;
      }
      .main {
        flex: 1;
      }
      .left,.right {
        width: 200px;
      }
    </style>
  </head>
  <body>
    <div class="container">
      <div class="left">left</div>
      <div class="main">main</div>
      <div class="right">right</div>
    </div>
  </body>
</html>
  • Float 布局：
html
<html>
  <head>
    <style>
      .container {
        width: 100%;
      }
      .left{
        width: 200px;
        float: left;
      }
      .main {
        margin-left: 200px; /*main的左边距是left的宽度*/
        margin-right: 200px; /*main的右边距是right的宽度*/
      }
      .right {
        width: 200px;

    <div class="container">
      <div class="main">main</div>
      <div class="left">left</div>
      <div class="right">right</div>

  </body>

6. calc() 方法 ​

calc() 方法用于计算一个表达式，并返回一个计算后的值。主要用于解决以下问题

  • 响应式布局：calc() 可以根据屏幕宽度动态计算宽度，实现响应式布局。
  • 动态尺寸调整：calc() 可以根据其他元素的尺寸或动态内容的大小来计算元素的尺寸
  • 优化代码：calc() 可以减少不必要的CSS代码和样式属性的硬编码，提高代码的效率。

8. 渐进增强和优雅降级 ​

  • 渐进增强（Progressive Enhancement）：

    • 先构建基本的功能，满足大多数用户的基本需求
    • 然后针对支持新特性的高级浏览器，逐步添加更多增强功能
    • 核心理念：让内容和基础功能可以被所有用户访问

  • 优雅降级（Graceful Degradation）：

    • 先构建完整的功能，针对高级浏览器进行开发
    • 然后针对低版本浏览器进行特殊处理，保证基本功能可用
    • 核心理念：让网站在任何环境下都能正常运行，但功能和体验可能有所降低

9. iframe 有哪些优缺点及使用场景 ​

iframe（内联框架）是一个 HTML 元素，它允许将另一个 HTML 页面嵌入到当前页面中。

  • 基础用法
html
<iframe 
  src="要嵌入的页面URL" 
  width="宽度" 
  height="高度"
  frameborder="0" 
  scrolling="auto"
  sandbox="allow-same-origin allow-scripts"
></iframe>
  • 优点

  1. 隔离性

    • 独立的上下文环境，不会与主页面 JavaScript 和 CSS 冲突
    • 可以加载独立的第三方内容而不影响主站点
    • 适合沙箱环境

  2. 复用性

    • 可以在多个页面中复用相同的内容
    • 适合展示导航栏、页脚等公共组件
    • 修改一处，多处同步更新

  3. 局部刷新

    • 可以独立刷新框架内容，不影响主页面
    • 适合实现类似后台管理系统的局部更新
  • 缺点

  1. 性能问题

    • 每个 iframe 都需要额外的 HTTP 请求
    • 会阻塞主页面的 onload 事件
    • 占用额外的内存和系统资源

  2. SEO 不友好

    • 搜索引擎可能无法正确索引 iframe 中的内容
    • 不利于网站的搜索引擎优化

  3. 安全性考虑

    • 可能存在点击劫持风险
    • 跨域通信需要特殊处理
    • 需要注意同源策略限制

  4. 用户体验

    • 移动端适配困难
    • 可能出现双滚动条
    • 浏览器的后退按钮可能失效
  • 常见使用场景
  1. 后台管理系统
html
<!-- 典型的后台管理布局 -->
<div class="admin-layout">
  <div class="sidebar">导航菜单</div>
  <div class="main-content">
    <iframe id="mainFrame" src="default.html" name="mainFrame"></iframe>
  </div>
</div>
  1. 第三方内容集成
html
<!-- 嵌入第三方地图 -->
<iframe 
  src="https://maps.google.com/maps?..." 
  width="600" 
  height="450" 
  frameborder="0" 
  allowfullscreen
></iframe>
  1. 广告展示
html
<!-- 广告iframe，设置sandbox限制权限 -->
<iframe 
  src="ad-content.html" 
  sandbox="allow-scripts allow-same-origin"
  width="300"
  height="250"
></iframe>
  • 安全处理
  1. 设置 CSP（Content Security Policy）
html
<meta 
  http-equiv="Content-Security-Policy" 
  content="frame-src 'self' https://trusted-site.com"
>
  1. 防止点击劫持
html
<!-- 服务器响应头 -->
X-Frame-Options: SAMEORIGIN

<!-- 或者使用 CSP -->
Content-Security-Policy: frame-ancestors 'self'
  1. 跨域通信
javascript
// 父页面发送消息
iframe.contentWindow.postMessage('hello', 'http://allowed-domain.com');

// iframe接收消息
window.addEventListener('message', function(event) {
  if (event.origin !== 'http://allowed-domain.com') return;
  console.log('收到消息:', event.data);
});
  • 性能优化建议
  1. 延迟加载
javascript
// 当页面主要内容加载完成后再加载iframe
window.addEventListener('load', function() {
  const iframe = document.createElement('iframe');
  iframe.src = 'heavy-content.html';
  document.body.appendChild(iframe);
});
  1. 使用 loading 属性
html
<iframe src="content.html" loading="lazy"></iframe>
  1. 合理设置大小
html
<iframe 
  width="100%" 
  height="100%" 
  style="min-height: 500px;"
  onload="this.style.height=this.contentDocument.body.scrollHeight+'px'"
></iframe>

10. CSS盒子模型 ​

  • 标准盒子模型（content-box）

    • width/height = content
    • 盒子总宽度 = width + padding + border + margin
    • 默认值，浏览器默认使用这个盒模型

  • IE盒子模型（border-box）

    • width/height = content + padding + border
    • 盒子总宽度 = width + margin
    • 更直观，常用于实际开发
css
/* 设置盒模型类型 */
.box {
  box-sizing: content-box;  /* 标准盒模型 */
  box-sizing: border-box;   /* IE盒模型 */
}

/* 常见的全局设置 */
* {
  box-sizing: border-box;
}

  • 应用场景

    • content-box：需要精确控制内容区域大小时
    • border-box：布局时希望padding和border包含在width中

  • 获取盒模型宽高

javascript
// 元素宽高（不包含padding和border）
element.clientWidth
element.clientHeight

// 元素宽高（包含padding和border）
element.offsetWidth
element.offsetHeight

// 元素内容实际宽高（包含溢出部分）
element.scrollWidth
element.scrollHeight

13. CSS 中元素的继承性 ​

CSS中的继承性指子元素会自动继承父元素的某些样式属性。

  • 会继承的属性

    • 字体相关: font-family、font-size、font-weight、font-style等
    • 文本相关: text-align、text-indent、line-height、color等
    • 可见性: visibility
    • 光标: cursor

  • 不会继承的属性

    • 盒模型相关: width、height、margin、padding、border等
    • 定位相关: position、top、right、bottom、left等
    • 背景相关: background、background-color等
    • 布局相关: display、float、clear等

  • 继承控制

css
/* 强制继承 */
.child {
  color: inherit;  /* 强制继承父元素的color */
}

/* 禁止继承 */
.child {
  color: initial;  /* 使用浏览器默认值 */
}

/* 使用未设置值 */
.child {
  color: unset;    /* 如果有继承值就使用继承值,否则使用初始值 */
}
  • 实际应用示例
css
/* 常见的全局文字设置 */
body {
  font-family: Arial, sans-serif;
  line-height: 1.5;
  color: #333;
}

/* 链接不继承颜色 */
a {
  color: #0066cc;  /* 需要单独设置 */
}

/* 标题继承字体但不继承大小 */
h1, h2, h3 {
  font-size: 2em;  /* 需要单独设置 */
  font-family: inherit;  /* 继承body的字体 */
}

14. CSS 优先级规则 ​

CSS优先级按照特殊性(Specificity)来计算,特殊性越高,优先级越高。

  • 优先级顺序(从高到低)

    1. !important
    2. 内联样式(style属性)
    3. ID选择器(#id)
    4. 类选择器(.class)、属性选择器([type="text"])、伪类(:hover)
    5. 标签选择器(div)、伪元素(::before)
    6. 通配符(*)、关系选择器(>、+)

  • 权重计算

    • 内联样式: 1000
    • ID选择器: 100
    • 类/属性/伪类: 10
    • 标签/伪元素: 1
    • 通配符: 0
css
/* 权重示例 */
#header .nav li { }           /* 权重: 100 + 10 + 1 = 111 */
.nav .list li.active { }      /* 权重: 10 + 10 + 1 + 10 = 31 */
div > p { }                   /* 权重: 1 + 1 = 2 */

/* !important的使用 */
.important-text {
  color: red !important;      /* 最高优先级 */
}

16. 什么是浮动，浮动会引起什么问题，有何解决方案？ ​

  • 浮动的定义：

    • float 属性使元素脱离文档流，向左或向右浮动
    • 浮动元素会尽量靠近容器的左侧或右侧
    • 文本和行内元素会环绕浮动元素

  • 浮动引起的问题：

    • 父元素高度塌陷：无法自动撑开包含浮动元素的高度
    • 后续元素布局受影响：会受到浮动元素的影响而上移

  • 清除浮动的方案：

css
/* 方案1: 使用 clear 属性 */
.clear { clear: both; }

/* 方案2: 父元素设置 overflow */
.parent { overflow: hidden; }

/* 方案3: 使用伪元素(推荐) */
.clearfix::after {
  content: "";
  display: block;
  clear: both;
}

17. line-height 和 height 的区别 ​

  • height: 元素的整体高度
  • line-height: 行高,即文本行基线之间的距离
  • 区别:
    • height是元素的实际高度,line-height是每行文字的高度
    • 单行文字垂直居中可以设置line-height等于height
    • line-height可以是数字(如1.5),表示字体大小的倍数

18. 为什么 img 是 inline 但是可以设置宽高？ ​

  • img 是 可替换元素，类似的还有 video、iframe、input 等
  • 可替换元素具有内在尺寸，可以设置 width 和 height
  • 虽然是 inline，但表现类似于 inline-block

19. box-sizing 的作用，如何使用？ ​

  • 作用: 用于控制元素的盒模型类型
  • 取值:
    • content-box: 标准盒模型,width/height 只包含内容区
    • border-box: IE盒模型,width/height 包含内容区、内边距和边框
  • 使用建议: 通常在全局设置 border-box
css
* {
  box-sizing: border-box;
}

20. Less 和 Sass 的区别 ​

  • 语法差异

    • Less 使用 @ 定义变量,Sass 使用 $
    • Less 完全兼容 CSS 语法,Sass 有专属语法(但也支持 CSS)
    • Sass 支持条件语句,Less 不支持

  • 编译环境

    • Less 基于 JavaScript,浏览器可直接编译
    • Sass 基于 Ruby,需要在服务器端编译

  • 使用建议

    • Less: 入门简单,适合小型项目
    • Sass: 功能强大,适合大型项目,生态更完善

23. link 标签和 import 标签的区别 ​

  • 加载顺序： link 并行加载，@import 串行加载
  • DOM可控性： link 可以通过 JS 操作，@import 不能
  • 兼容性： link 兼容性好，@import 仅 IE5+ 支持
  • 加载时机： link 加载时就解析，@import 页面加载完再执行

24. 块元素、行内元素、置换元素 ​

  • 块元素： 独占一行，可设置宽高，如 div、p、h1
  • 行内元素： 不换行，不可设置宽高，如 span、a
  • 置换元素： 具有内在尺寸，可设置宽高的行内元素，如 img、input

27. 双冒号和单冒号 ​

  • 单冒号(😃： CSS2 的伪类，如 :hover、:active
  • 双冒号(:😃： CSS3 的伪元素，如 ::before、::after
  • 兼容性： 为兼容旧浏览器，::before 可写成 :before
