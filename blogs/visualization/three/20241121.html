前言 🌈 ​

在开发 Three.js 项目时,性能优化是一个永恒的话题。本文将结合实际项目经验,详细介绍 Three.js 性能优化的各个方面,并提供具体的优化方案和代码示例。

场景介绍 🎮 ​

假设我们正在开发一个大型的3D可视化项目,包含以下场景:

  • 城市建筑模型展示
  • 大量动态粒子效果
  • 实时数据更新
  • 多个光源和阴影
  • 后期处理效果

1. 几何体优化 📐 ​

1.1 合并几何体 ​

当场景中有大量相似的几何体时,可以使用合并来减少 draw calls。

js
// 优化前
buildings.forEach(building => {
  const geometry = new THREE.BoxGeometry(1, 1, 1);
  const mesh = new THREE.Mesh(geometry, material);
  scene.add(mesh);
});
// 优化后
const geometries = [];
buildings.forEach(building => {
  const geometry = new THREE.BoxGeometry(1, 1, 1);
  geometry.translate(building.x, building.y, building.z);
  geometries.push(geometry);
});
const mergedGeometry = BufferGeometryUtils.mergeBufferGeometries(geometries);
const mesh = new THREE.Mesh(mergedGeometry, material);
scene.add(mesh);

1.2 使用 InstancedMesh ​

对于大量相同的对象,使用 InstancedMesh 可以显著提升性能。

js
// 优化前: 10000个立方体
for (let i = 0; i < 10000; i++) {
  const mesh = new THREE.Mesh(geometry, material);
  scene.add(mesh);
}
// 优化后
const instancedMesh = new THREE.InstancedMesh(
  geometry,
  material,
  10000
);
const matrix = new THREE.Matrix4();
for (let i = 0; i < 10000; i++) {
  matrix.setPosition(
    Math.random() * 100 - 50,
    Math.random() * 100 - 50,
    Math.random() * 100 - 50,
  );
  instancedMesh.setMatrixAt(i, matrix);
}
scene.add(instancedMesh);

1.3 LOD (Level of Detail) ​

根据距离使用不同精度的模型。

js
const lod = new THREE.LOD();
// 高精度模型 (近距离)
const highDetailGeometry = new THREE.BoxGeometry(1, 1, 1, 32, 32, 32);
const highDetailMesh = new THREE.Mesh(highDetailGeometry, material);
lod.addLevel(highDetailMesh, 0);
// 中等精度模型 (中距离)
const mediumDetailGeometry = new THREE.BoxGeometry(1, 1, 1, 16, 16, 16);
const mediumDetailMesh = new THREE.Mesh(mediumDetailGeometry, material);
lod.addLevel(mediumDetailMesh, 50);
// 低精度模型 (远距离)
const lowDetailGeometry = new THREE.BoxGeometry(1, 1, 1, 8, 8, 8);
const lowDetailMesh = new THREE.Mesh(lowDetailGeometry, material);
lod.addLevel(lowDetailMesh, 100);
scene.add(lod);

7.2 使用对象池 ​

js
class ParticlePool {
  constructor(size) {
    this.pool = [];
    this.activeParticles = new Set();
    
    // 预创建粒子
    for (let i = 0; i < size; i++) {
      this.pool.push(this.createParticle());
    }
  }
  
  createParticle() {
    return new THREE.Mesh(
      new THREE.SphereGeometry(0.1),
      new THREE.MeshBasicMaterial()
    );
  }
  
  get() {
    const particle = this.pool.pop();
    if (particle) {
      this.activeParticles.add(particle);
      return particle;
    }
    return this.createParticle();
  }
  
  release(particle) {
    this.activeParticles.delete(particle);
    this.pool.push(particle);
  }
}

8.2 自定义性能监控 ​

js
class PerformanceMonitor {
  constructor() {
    this.fps = 0;
    this.frames = 0;
    this.lastTime = performance.now();
  }
  
  update() {
    const currentTime = performance.now();
    this.frames++;
    
    if (currentTime >= this.lastTime + 1000) {
      this.fps = (this.frames * 1000) / (currentTime - this.lastTime);
      this.lastTime = currentTime;
      this.frames = 0;
      
      console.log(`FPS: ${this.fps.toFixed(1)}`);
    }
  }
}

总结 📝 ​

Three.js 性能优化是一个系统工程,需要从多个层面进行优化:

  • 模型层面：合并、实例化、LOD
  • 渲染层面：配置优化、缓冲区管理
  • 资源层面：纹理压缩、预加载
  • 运行时：内存管理、动画优化

最重要的是要根据实际项目需求和性能瓶颈,选择合适的优化策略。
