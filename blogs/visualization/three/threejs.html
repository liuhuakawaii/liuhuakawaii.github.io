概念 ​

前置知识 ​

  • 场景(Scene)：是物体、光源等元素的容器，可以配合 chrome 插件使用，抛出 window.scene即可实时调整 obj 的信息和材质信息。
  • 相机（Camera）：场景中的相机，代替人眼去观察，场景中只能添加一个，一般常用的是透视相机（PerspectiveCamera）
  • 物体对象（Mesh）：包括二维物体（点、线、面）、三维物体，模型等等
  • 光源（Light）：场景中的光照，如果不添加光照场景将会是一片漆黑，包括全局光、平行光、点光源等
  • 渲染器（Renderer）:场景的渲染方式，如webGL\canvas2D\Css3D。
  • 控制器(Control): 可通过键盘、鼠标控制相机的移动

相机(Camera) ​

  • 正交（orthographic）相机：物品的渲染尺寸与它距离镜头的远近无关。也就是说在场景中移动一个物体，其大小不会变化。正交镜头适合2D游戏。
  • 透视（perspective）相机：模拟人眼的视觉特点，距离远的物体显得更小。透视镜头通常更适合3D渲染
  • THREE.PerspectiveCamera(fov,aspect,near,far)
    • fov视野角度：FOV取值在60-90度之间较好，默认值为60
    • aspect渲染区域的宽高比：一般是rendder.domElement.clientWidth/render.domElement.clientHeight
    • near最近离镜头的距离
    • far 远离镜头的距离
  • camera.position：控制相机在整个3D环境中的位置
  • camera.lookAt：控制相机的焦点位置，决定相机的朝向

渲染器（renderer） ​

  • 设置渲染器的大小：renderer.setSize(width,height)
  • 设置启用抗锯齿: renderer.antialias = true
  • 设置设备像素比: renderer.setPixelRatio(window.devicePixelRatio)
  • 设置背景颜色: renderer.setClearColor(0x444444, 1);
  • 开始渲染：renderer.render(scene,camera)
  • 获取渲染元素：renderer.domElement，本质是<canvas>

光源 ​

光源说明
AmbientLight环境光，其颜色均匀的应用到场景及其所有对象上,这种光源为场景添加全局的环境光。这种光没有特定的方向，不会产生阴影。通常不会把AmbientLight作为唯一的光源，而是和SpotLight、DirectionalLight等光源结合使用，从而达到柔化阴影、添加全局色调的效果。指定颜色时要相对保守，例如#0c0c0c。设置太亮的颜色会导致整个画面过度饱和，什么都看不清：
PointLight3D空间中的一个点光源，向所有方向发出光线
SpotLight产生圆锥形光柱的聚光灯，台灯、天花板射灯通常都属于这类光源,这种光源的使用场景最多，特别是在你需要阴影效果的时候。
DirectionalLight也就无限光，光线是平行的。典型的例子是日光,用于模拟遥远的，类似太阳那样的光源。该光源与SpotLight的主要区别是，它不会随着距离而变暗，所有被照耀的地方获得相同的光照强度。
HemisphereLight特殊光源，用于创建户外自然的光线效果，此光源模拟物体表面反光效果、微弱发光的天空,模拟穹顶（半球）的微弱发光效果，让户外场景更加逼真。使用DirectionalLight + AmbientLight可以在某种程度上来模拟户外光线，但是不够真实，因为无法体现大气层的散射效果、地面或物体的反射效果
AreaLight面光源，指定一个发光的区域
LensFlare不是光源，用于给光源添加镜头光晕效果

加载外部模型 ​

  • 通过Three.js加载器（Loader）实现
  • 加载器把文本/二进制的模型文件转化为Three.js对象结构。 每个加载器理解某种特定的文件格式。
  • 需要注意的是，由于贴图的尺寸必须是(2的幂数)X (2的幂数)，如：1024X512，所以为了防止贴图变形，平面的宽度比例需要与贴图的比例一致。
  • 支持的格式
    格式说明
    JSONThree.js自定义的、基于JSON的格式。可以声明式的定义一个Geometry或者Scene.利用该格式，你可以方便的重用复杂的Geometry或Scene
    OBJ / MTLOBJ是Wavefront开发的一种简单3D格式，此格式被广泛的支持，用于定义Geometry，MTL用于配合OBJ，它指定OBJ使用的材质
    Collada（dae）基于XML的格式，被大量3D应用程序、渲染引擎支持
    STLSTereoLithography的简写，在快速原型领域被广泛使用。3D打印模型通常使用该格式定义Three.js提供了STLExporter.js，使用它可以把Three.js模型导出为STL格式
    CTMopenCTM定义的格式，以紧凑的格式存储基于三角形的Mesh
    VTKVisualization Toolkit定义的格式，用于声明顶点和面。此格式有二进制/ASCII两种变体，Three.js仅支持ASCII变体
    AWD3D场景的二进制格式，主要被away3d引擎使用，Three.js不支持AWD压缩格式
    Assimp开放资产导入库（Open asset import library）是导入多种3D模型的标准方式。使用该Loader你可以导入多种多样的3D模型格式
    VRML虚拟现实建模语言（Virtual Reality Modeling Language）是一种基于文本的格式，现已经被X3D格式取代尽管Three.js不直接支持X3D，但是后者很容易被转换为其它格式
    Babylon游戏引擎Babylon的私有格式
    PLY常用于存储来自3D扫描仪的信息

使用模板 ​

js
   class Scene {
      constructor(renderTarget) {
        this.scene = new THREE.Scene();
        this.camera = null;
        this.renderer = null;
        this.controls = null;
        this.mesh = null
        this.renderTarget = renderTarget; // 渲染目标

        this.initRenderer();
        this.initCamera();
        this.initControls();
        this.initLights();
        this.initHelper();
        this.addMesh()
        this.Stats = new Stats(); //性能监视器
        this.render();
        window.addEventListener('resize', this.onWindowResize.bind(this), false);
        document.body.appendChild(this.Stats.domElement);
      }

      initRenderer() {
        this.renderer = new THREE.WebGLRenderer({ antialias: true });
        this.renderer.setSize(this.renderTarget.clientWidth, this.renderTarget.clientHeight);
        this.renderer.setPixelRatio(window.devicePixelRatio); // 屏幕像素比
        this.renderer.setClearColor(0x444444, 1); // 背景颜色
        this.renderTarget.appendChild(this.renderer.domElement);
      }

      initCamera() {
        this.camera = new THREE.PerspectiveCamera(30, this.renderTarget.clientWidth / this.renderTarget.clientHeight, 1, 3000);
        this.camera.position.set(3, 3, 3);
        this.camera.lookAt(new THREE.Vector3(0, 0, 0));
      }

      initControls() {
        this.controls = new OrbitControls(this.camera, this.renderer.domElement);
        this.controls.enableDamping = true; // 启用阻尼
        this.controls.dampingFactor = 0.1; // 阻尼惯性
        this.controls.minDistance = 5;
        this.controls.maxDistance = 20;
        this.controls.target.set(0, 0, 0) //注意和lookAt保持一致
        this.controls.addEventListener('change', () => {
          this.renderer.render(this.scene, this.camera);
        });
        this.controls.zoomSpeed = 0.8; // 缩放速度
      }

      initLights() {
        const ambientLight = new THREE.AmbientLight(0x404040); // (color ,intensity) 光照强度
        this.scene.add(ambientLight);

        const pointLight = new THREE.PointLight(0xffffff, 0.1); // (color, intensity, distance, decay) 光照衰减
        pointLight.position.set(-3, -6, -5);
        this.scene.add(pointLight);

        const directionalLight = new THREE.DirectionalLight(0xffffff, 0.8);
        directionalLight.position.set(3, 6, 5);
        this.scene.add(directionalLight);
      }

      initHelper() { /* 辅助对象 */
        const size = 100; // 网格尺寸
        const divisions = 100; // 网格分段数
        const gridHelper = new THREE.GridHelper(size, divisions, 0xCCCCCC, 0xCCCCCC);
        gridHelper.material.opacity = 0.2;
        gridHelper.material.transparent = true;
        gridHelper.name = 'gridHelper'
        this.scene.add(gridHelper);

        const axesHelper = new THREE.AxesHelper(5);
        axesHelper.name = 'axesHelper'
        this.scene.add(axesHelper);
      }

      addMesh(){
        const geometry = new THREE.BoxGeometry(1, 1, 1);
        const material = new THREE.MeshLambertMaterial({ color: 0x00ff00 });
        this.mesh = new THREE.Mesh(geometry, material);
        this.scene.add(this.mesh);
      }

      onWindowResize() {
        // 渲染器执行render方法的时候会读取相机对象的投影矩阵属性projectionMatrix
        // 但是不会每渲染一帧，就通过相机的属性计算投影矩阵(节约计算资源)
        // 如果相机的一些属性发生了变化，需要执行updateProjectionMatrix ()方法更新相机的投影矩阵
        this.camera.aspect = this.renderTarget.clientWidth / this.renderTarget.clientHeight; //更新摄像头的宽高比
        this.camera.updateProjectionMatrix();//更新投影矩阵
        this.renderer.setSize(this.renderTarget.clientWidth, this.renderTarget.clientHeight);//重置渲染器输出画布canvas尺寸
        this.renderer.setPixelRatio(window.devicePixelRatio) //设置渲染器的像素比
        this.renderer.render(this.scene, this.camera);
      }

      render() {
        requestAnimationFrame(this.render.bind(this)); // 周期性执行
        this.Stats.update()
        this.controls.update(); //update()函数内会执行camera.lookAt(controls.target)
        this.renderer.render(this.scene, this.camera); // 渲染场景,类似于拍照
      }
    }

object.rotation.set(Math.PI/4, 0, Math.PI/2);

// 获取欧拉角
let currentRotation = object.rotation;

const b = new THREE.Vector3( 1, 0, 1 );

四元数Quaternion ​

四元数用四个数值(x, y, z, w)来表示3D旋转。虽然不如欧拉角直观,但在数学运算和插值方面有优势。 欧拉角和四元数可以互相转换 基础使用

js
// 创建四元数
let quaternion = new THREE.Quaternion();

// 从欧拉角设置四元数
quaternion.setFromEuler(new THREE.Euler(Math.PI/4, 0, Math.PI/2));

// 应用四元数到物体
object.quaternion.copy(quaternion);

// 获取四元数
let currentQuaternion = object.quaternion;
const q = new THREE.Quaternion(0, 0, 0, 1); //四元数
q.setFromAxisAngle(axis, Math.PI/4); //绕axis轴旋转π/4
mesh.quaternion.multiply(q); //绕四元数旋转
mesh.rotateOnWorldAxis(axis,Math.PI/8); //绕世界轴旋转

注意事项

  • 确保使用的是单位四元数(长度为1)。Three.js通常会自动处理这个问题。
  • 使用 .slerp() 方法进行平滑的旋转插值。
  • 避免直接修改四元数的 x, y, z, w 值:最好使用提供的方法如 .setFromEuler(), .setFromAxisAngle() 等。 选择建议
  • 如果需要直观地设置或显示旋转,使用欧拉角。
  • 对于动画、插值或需要进行大量旋转计算的情况,使用四元数。
  • 在存储或传输旋转数据时,考虑使用四元数以避免万向节死锁问题。（当其中一个旋转轴与另一个旋转轴重合时,会失去一个自由度,这就是万向节死锁。）

MeshSurfaceSampler ​

MeshSurfaceSampler是Three.js库中提供的一个工具，用于从三维模型表面均匀地抽样位置。这种方法主要用于散点生成、粒子效果和任何需要在模型表面随机分布元素的场合。

原理 ​

MeshSurfaceSampler的工作原理基于对三角形网格模型表面的采样。简单来说，它会根据模型的面积分布，在模型表面随机抽取点。这一过程包含几个关键步骤：

  1. **计算每个三角形的面积：**首先，MeshSurfaceSampler会遍历模型的所有三角形，计算它们的面积。这一步骤是基于三角形顶点坐标来完成的。

  2. **构建累积面积数组：**随后，它会构建一个累积面积数组，数组中每个元素的值等于从第一个三角形到当前三角形的所有面积之和。这样可以根据三角形面积在模型上创建一个概率分布，从而确保点的抽样与三角形的面积成比例。

  3. **随机抽样：**当需要生成一个点时，MeshSurfaceSampler会生成一个随机数，这个随机数对应于累积面积数组中的某个值。通过比较随机数与累积面积数组的元素值，可以选择对应的三角形进行采样。

  4. **在选定的三角形内生成点：**一旦选定了三角形，MeshSurfaceSampler会在这个三角形内随机生成一个点。这一步通常涉及到在三角形内使用重心坐标或其他方法来保证点的均匀分布。

用法 ​

calculateSurfaceArea(mesh) {
    const geometry = mesh.geometry;
    let area = 0;
    const vertices = geometry.attributes.position.array;
    const indices = geometry.index ? geometry.index.array : undefined;
    for (let i = 0; i < (indices ? indices.length : vertices.length / 3); i += 3) {
    const id0 = indices ? indices[i] * 3 : i * 3;
    const id1 = indices ? indices[i + 1] * 3 : (i + 1) * 3;
    const id2 = indices ? indices[i + 2] * 3 : (i + 2) * 3;

    const v0 = new THREE.Vector3(vertices[id0], vertices[id0 + 1], vertices[id0 + 2]);
    const v1 = new THREE.Vector3(vertices[id1], vertices[id1 + 1], vertices[id1 + 2]);
    const v2 = new THREE.Vector3(vertices[id2], vertices[id2 + 1], vertices[id2 + 2]);

    // 通过计算三角形面积来计算总面积
    const edge1 = new THREE.Vector3().subVectors(v1, v0);
    const edge2 = new THREE.Vector3().subVectors(v2, v0);
    const crossProduct = new THREE.Vector3().crossVectors(edge1, edge2);
    area += crossProduct.length() * 0.5;
    }
    return area;
}

        return { scaleFactor: scaleFactor, center: center };
    }

缩放 ​

在 Three.js 中，你可以使用 Vector3 类对向量进行缩放。以下是有关 Vector3 缩放的方法，包括如何独立缩放各个轴：

  1. 统一缩放 使用 multiplyScalar 方法，可以将向量的三个分量（x, y, z）按相同的比例缩放。 multiplyScalar(scalar: number): Vector3：将每个分量乘以指定的标量。
js
const vector = new THREE.Vector3(1, 2, 3);
const scaleFactor = 2;
vector.multiplyScalar(scaleFactor);
console.log(vector); // Vector3 { x: 2, y: 4, z: 6 }
  1. 非统一缩放 使用 multiply 方法，可以将向量的各个分量按不同的比例进行缩放。 multiply(v: Vector3): Vector3：将每个分量乘以另一个向量的对应分量。
js
const vector = new THREE.Vector3(1, 2, 3);
const scaleVector = new THREE.Vector3(2, 3, 4);
vector.multiply(scaleVector);
console.log(vector); // Vector3 { x: 2, y: 6, z: 12 }
  1. 手动设置每个分量 你也可以手动设置向量的每个分量来实现缩放。 set(x: number, y: number, z: number): Vector3：将向量的分量设置为指定的值。
js
const vector = new THREE.Vector3(1, 2, 3);
const scaleX = 2;
const scaleY = 3;
const scaleZ = 4;
vector.set(vector.x * scaleX, vector.y * scaleY, vector.z * scaleZ);
console.log(vector); // Vector3 { x: 2, y: 6, z: 12 }
  1. 使用 applyMatrix4 方法 对于更复杂的变换（包括缩放），你可以使用变换矩阵。 applyMatrix4(matrix: Matrix4): Vector3：将4x4变换矩阵应用到向量上。
js
const vector = new THREE.Vector3(1, 2, 3);
const matrix = new THREE.Matrix4();
matrix.makeScale(2, 3, 4);
vector.applyMatrix4(matrix);
console.log(vector); // Vector3 { x: 2, y: 6, z: 12 }

手动生成圆弧顶点 ​

生成圆弧顶点数据 以坐标原点为中心，在XOY平面上生成圆弧上的顶点数据。

绘制圆弧线，本质就是绘制一个正n边形，n越大，圆弧细分数或者说精度越高。

通过for循环沿着圆弧线，通过三角函数计算顶点坐标，批量生成圆弧上顶点数据。

js
const geometry = new THREE.BufferGeometry(); //创建一个几何体对象
const R = 1; //圆弧半径
const N = 100; //分段数量
const sp = 2 * Math.PI / N;//两个相邻点间隔弧度
const arr = [];
const cx = 0;
const cy = 0;
for (let i = 0; i < N; i++) {
    const angle = sp * i;//当前点弧度 以(cx,cy)为中心，在XOY平面上生成圆弧上的顶点数据 
    const x = cx + R * Math.cos(angle);
    const y = cy + R * Math.sin(angle); 
    arr.push(x, y, 0);
}
const vertices = new Float32Array(arr); // 创建属性缓冲区对象 //3个为一组，表示一个顶点的xyz坐标 
const attribue = new THREE.BufferAttribute(vertices, 3); // 设置几何体attributes属性的位置属性
geometry.attributes.position = attribue; 
const material = new THREE.LineBasicMaterial({ // 线材质  
    color: 0xffffff 
}); // 创建线模型对象 构造函数：Line、LineLoop、LineSegments  
// const line=new THREE.Line(geometry, material); 
const line = new THREE.LineLoop(geometry, material);//线条模型对象
this.scene.add(line);

注意事项

  1. 阴影贴图分辨率：
    • 更高的分辨率可以产生更清晰的阴影，但也会影响性能。
    • 如果发现阴影模糊或有锯齿，可以尝试增加分辨率。
  2. 投影范围：
    • 设置投影相机的范围时要考虑到场景的大小，以确保所有的阴影都被正确捕捉。
    • 如果投影范围设置得过小，可能会导致某些阴影丢失。
    • 如果设置得过大，则可能导致阴影细节丢失。
  3. 阴影偏移：
    • 设置正确的偏移值可以减少自阴影问题，即物体自身对其表面产生阴影的情况。
    • 偏移值通常很小，例如 -0.0001，以避免物体穿透自身的阴影。
  4. 阴影过滤：
    • 可以使用 PCF (Percentage Closer Filtering) 或者 PCF Soft 来改善阴影边缘的质量。
    • 通过设置 THREE.Light.prototype.shadow.mapNeedsUpdate 为 true 来启用。
  5. 性能考虑：
    • 大量使用阴影可能会降低渲染性能，尤其是在移动设备上。
    • 如果发现性能瓶颈，可以考虑减少阴影贴图的分辨率，或者只对某些重要的光源启用阴影。

模型表面一闪一闪（z-fighting/深度冲突问题） ​

这种现象发生在两个或多个表面非常接近或重叠时，渲染器难以确定哪个表面应该显示在前面

解决方案：

  1. 重叠几何体：
    • 检查模型，确保没有重叠的几何体。
    • 如果有意为之的重叠，可以稍微调整几何体的位置，使它们有微小的间隔。
  2. 相机近平面和远平面设置不当：
    • 调整相机的near和far值，使其更适合你的场景尺寸：
    js
        this.camera = new THREE.PerspectiveCamera(
        30,
        this.container.clientWidth / this.container.clientHeight,
        1,  // 增加近平面距离
        3000
    );
  3. 深度缓冲精度不足：
    • 对于大场景，默认的深度缓冲可能不够精确。
    js
        this.renderer = new THREE.WebGLRenderer({ 
        antialias: true,
        logarithmicDepthBuffer: true  // 添加这一行
    });
  4. 材质问题：

        object.material.polygonOffset = true;

        object.material.side = THREE.FrontSide;  // 或 THREE.BackSide
