Framer-motion使用 ​

入门介绍 ​

基本组件：motion ​

Framer Motion 的核心是 motion 组件。它可以应用于任何 HTML 或 SVG 元素。

js
import { motion } from "framer-motion"

<motion.div />
<motion.span />
<motion.svg />

属性追踪：MotionValues ​

MotionValues用于在 React 渲染生命周期之外跟踪动画值的状态和速度。

js
export const MyComponent = () => {
  const x = useMotionValue(0)
  const background = useTransform(
    x,
    [-100, 0, 100],
    ["#ff008c", "#7700ff", "rgb(230, 255, 0)"]
  )

  return (
    <motion.div style={{ background }}>
      <motion.div
        drag="x"
        dragConstraints={{ left: 0, right: 0 }}
        style={{ x }}
      >
        <Icon x={x} />
      </motion.div>
    </motion.div>
  )
}

动画属性: Animations ​

  1. animate：定义元素的目标状态。
js
// 这会使元素在 x 轴上移动 100px 并淡出。
<motion.div animate={{ x: 100, opacity: 0 }} />

关键帧

  • 将值设置为一个数组，Motion 将依次通过每个值进行动画处理。
  • 可以通过传递 null 来表示当前值作为初始关键帧
  • 默认情况下，每个关键帧将在整个动画中均匀分布，times 是一个与关键帧数组长度相同的数组，其中的数字0和1定义了动画中每个关键帧应该被击中的位置。
js
import { motion } from "framer-motion"

export const MyComponent = () => (
  <motion.div
    animate={{
      scale: [1, 2, 2, 1, 1],
      rotate: [0, 0, 270, 270, 0],
      borderRadius: ["20%", "20%", "50%", "50%", "20%"],
      whileHover={{ scale: [null, 1.5, 1.4] }}
      transition={{ duration: 3, times: [0, 0.2, 1] }}
    }}
  />
)
  1. initial：定义元素的初始状态。
js
// 元素会从不可见状态淡入。
<motion.div initial={{ opacity: 0 }} animate={{ opacity: 1 }} />
  1. transition：定义动画的过渡特性。
js
// 这个动画会持续 2 秒，使用 "easeOut" 缓动函数。
<motion.div
  animate={{ x: 100 }}
  transition={{ duration: 2, ease: "easeOut" }}
/>
  1. variants：
  • 定义可重用的动画状态。通过为组件及其子组件提供 variants 匹配的名称，可以通过更改单个 prop 来为整个 React 树设置动画。
  • variants 也可以是动态函数，根据传递给每个组件的 custom prop 的数据返回不同的 props。
js
import { motion } from "framer-motion"

const variants = {
 open: (height = 1000) => ({
    clipPath: `circle(${height * 2 + 200}px at 40px 40px)`,
    transition: {
      type: "spring",
      stiffness: 20,
      restDelta: 2
    }
  }),
  closed: {
    clipPath: "circle(30px at 40px 40px)",
    transition: {
      delay: 0.5,
      type: "spring",
      stiffness: 400,
      damping: 40
    }
  }
}

export const MyComponent = () => {
  const [isOpen, setIsOpen] = useState(false)

  return (
    <motion.nav
      animate={isOpen ? "open" : "closed"}
      variants={variants}
    >
      <Toggle onClick={() => setIsOpen(isOpen => !isOpen)} />
      <Items />
    </motion.nav>
  )
}
  • 默认情况下，所有这些动画将同时启动。但通过使用变体，我们可以访问额外的transition道具，如when、delayChildren和，staggerChildren 这可以让父母协调子动画的执行。
js
const list = {
  visible: {
    opacity: 1,
    transition: {
      when: "beforeChildren",
      staggerChildren: 0.3,
    },
  },
  hidden: {
    opacity: 0,
    transition: {
      when: "afterChildren",
    },
  },
}
  • 通过传递字符串数​​组来定义多个变体。如果变体定义相同的值，则数组中较晚出现的变体将优先于数组中较早出现的变体。

手势识别: Gestures ​

  1. whileHover：定义鼠标悬停时的状态。
  2. whileTap：定义元素被点击时的状态。
  3. whileDrag:定义元素被拖动时的状态。
  4. whileFocus:定义元素获得焦点时的状态。
js
import { motion } from "framer-motion"

export const MyComponent = () => (
  <motion.button
    whileHover={{ scale: 1.1 }}
    whileTap={{ scale: 0.9 }}
    drag
  />
)

拖动 ​

  1. drag：使元素可拖动。
  2. dragElastic：限制在特定范围内,并且这种弹性的强度可以通过 dragElastic 道具来配置。可以直接传入某个元素的ref来限制
js
import { motion } from "framer-motion"

export const MyComponent = () => (
  <motion.div
    drag
    dragConstraints={{
      top: -50,
      left: -50,
      right: 50,
      bottom: 50,
    }}
  />
)

滚动相关 ​

  1. whileInView：它定义了当组件位于视口中时要动画的视觉状态。
js
<motion.div
  initial={{ opacity: 0 }}
  whileInView={{ opacity: 1 }}
  viewport={{ once: true }}
/>
  1. useScroll:该钩子提供了四个只读的 MotionValue，两个以像素为单位返回视口的 x/y 滚动位置，另外两个将其作为0和之间的进度值返回1。 可以使用这些MotionValue来声明性地驱动进度指示器或视差效果等功能。
js
import { motion, useScroll } from "framer-motion"

export const CircleIndicator = () => {
  const { scrollYProgress } = useScroll()

  return (
    <motion.path
      d="M 0, 20 a 20, 20 0 1,0 40,0 a 20, 20 0 1,0 -40,0"
      style={{ pathLength: scrollYProgress }}
    />
  )
}
  1. useViewportScroll：获取视口滚动信息。
js
import { useViewportScroll } from "framer-motion"

function Component() {
  const { scrollY } = useViewportScroll()
  return <motion.div style={{ y: scrollY }} />
}

退出动画 ​

在 React 中，一旦组件从 DOM 中移除，通常很难为其设置动画。

通过用 包装 motion 组件 AnimatePresence，它们可以使用一个 exit 属性，该属性可以定义一组值或一个变体名称，以便在被删除之前进行动画处理。

js
import { motion, AnimatePresence } from "framer-motion"

export const Slideshow = ({ image }) => (
 <AnimatePresence>
  {isVisible && (
    <motion.div
      initial={{ opacity: 0 }}
      animate={{ opacity: 1 }}
      exit={{ opacity: 0 }}
    />
  )}
</AnimatePresence>
)

可以结合 React-router-6 来实现路由切换动画

js
 <AnimatePresence mode="wait" initial={false}>
      {React.cloneElement(element, { key: location.pathname })}
    </AnimatePresence>

布局动画 ​

  1. layout：为了在组件的大小或位置发生变化时自动为其布局设置动画motion，请为其提供一个 layout prop。 无论是什么原因导致布局发生变化，无论是父级弹性框的方向、宽度、顶部/右侧等，动画本身都将通过变换来执行，以实现最佳性能。
js
<motion.div layout />
  1. layoutId：当添加新 motion 组件时，可以通过为它们提供相同的 layoutId 道具来自动从另一个组件进行动画处理。
js
<motion.div layoutId="shared-id" />

动画手动控制 ​

声明式动画是大多数 UI 交互的理想选择。但有时我们需要编排更复杂的序列。 useAnimate:

  • 为任何 HTML/SVG 元素制作动画
  • 创建复杂的动画序列
  • time使用、speed、play()和其他播放控件来控制动画pause()。
js
const MyComponent = () => {
  const [scope, animate] = useAnimate()
  
  useEffect(() => {
    const animation = async () => {
      await animate(scope.current, { x: "100%" })
      animate("li", { opacity: 1 })
    }
    
    animation()
  }, [])

  return (
    <ul ref={scope}>
      <li />
      <li />
      <li />
    </ul>
  )
}

动画序列 ​

使用 async/await 和 controls.start() 创建动画序列：

js
const sequence = async () => {
  await controls.start({ x: 100 })
  await controls.start({ y: 100 })
  await controls.start({ opacity: 0 })
}

路径动画 ​

为 SVG 路径创建动画：

js
<motion.path
  d="M0 0 L100 100"
  initial={{ pathLength: 0 }}
  animate={{ pathLength: 1 }}
/>

SVG相关 ​

pathLength ：利用许多 SVG 元素上的特殊、pathSpacing和属性，可以简化线条绘制动画pathOffset。

js
<motion.circle
  initial={{ pathLength: 0 }}
  animate={{ pathLength: 1 }}
/>

优秀使用案例 ​

页面转场动画 ​

js
const pageVariants = {
  initial: { opacity: 0, x: "-100vw" },
  in: { opacity: 1, x: 0 },
  out: { opacity: 0, x: "100vw" }
};

const pageTransition = {
  type: "tween",
  ease: "anticipate",
  duration: 0.5
};

function Page() {
  return (
    <motion.div
      initial="initial"
      animate="in"
      exit="out"
      variants={pageVariants}
      transition={pageTransition}
    >
      {/* 页面内容 */}
    </motion.div>
  );
}

交互式卡片翻转 ​

js
const flipVariants = {
  front: { rotateY: 0 },
  back: { rotateY: 180 }
};

function FlipCard() {
  const [isFlipped, setIsFlipped] = useState(false);

  return (
    <motion.div
      animate={isFlipped ? "back" : "front"}
      variants={flipVariants}
      transition={{ duration: 0.6 }}
      onClick={() => setIsFlipped(!isFlipped)}
      style={{ width: 200, height: 300 }}
    >
      {/* 卡片内容 */}
    </motion.div>
  );
}

滚动触发动画 ​

js
import { useInView } from "react-intersection-observer";

function ScrollAnimation() {
  const [ref, inView] = useInView({
    threshold: 0.1,
    triggerOnce: true
  });

  return (
    <motion.div
      ref={ref}
      initial={{ opacity: 0, y: 50 }}
      animate={inView ? { opacity: 1, y: 0 } : {}}
      transition={{ duration: 0.5 }}
    >
      {/* 内容 */}
    </motion.div>
  );
}

原理解析 ​

  1. Framer Motion 使用 React 的 useRef 和 useEffect 钩子来管理动画状态和生命周期。
  2. 它利用 CSS transforms 和 opacity 属性来创建流畅的动画，这些属性在现代浏览器中有很好的性能表现。
  3. Framer Motion 内部使用 requestAnimationFrame 来同步动画帧，确保动画流畅。
  4. 对于复杂的动画序列，Framer Motion 使用 Promise 来管理动画的时序。
  5. 布局动画（layout 属性）使用 FLIP（First, Last, Invert, Play）技术来实现高性能的布局变化动画。
  6. 手势识别（如 drag）使用 Pointer 事件（如果可用）或回退到 Mouse 和 Touch 事件来实现跨设备兼容性。

使用建议 ​

  1. 尽量使用 variants 来定义动画状态，这样可以使代码更加清晰和可维护。
  2. 对于复杂的动画，考虑使用 useAnimation 钩子来获得更精细的控制。
  3. 利用 Framer Motion 的自动化特性，如 layout 属性，可以大大简化复杂布局变化的动画实现。
  4. 在处理大量动画元素时，考虑使用 AnimatePresence 来管理元素的进入和退出动画。
  5. 利用 Framer Motion 的 useTransform 和 useViewportScroll 等钩子来创建高级的交互效果。
