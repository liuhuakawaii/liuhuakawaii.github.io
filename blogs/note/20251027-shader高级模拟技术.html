🧪 Shader 进阶 - 高级模拟技术与交互系统 ​

🌟 突破Shader的限制 - 状态保存的挑战 ​

在之前的文章中,所有效果都是无状态的 (Stateless):

glsl
// 典型的无状态Shader
void main() {
    vec2 uv = gl_FragCoord.xy / u_resolution.xy;
    float n = noise(uv + u_time);  // 只依赖输入,没有"记忆"
    gl_FragColor = vec4(vec3(n), 1.0);
}

问题: 每一帧完全独立,无法实现:

  • ❌ 物理模拟 (需要记住上一帧的位置/速度)
  • ❌ 细胞自动机 (需要知道邻居的历史状态)
  • ❌ 流体模拟 (需要保存速度场/压力场)

解决方案: 乒乓缓冲 (Ping-Pong Buffer) - 用纹理存储状态!

🏓 乒乓缓冲 (Ping-Pong Buffer) - GPU的记忆系统 ​

🎯 核心思想 ​

用两个纹理轮流充当"读取源"和"写入目标":

帧 N:                          帧 N+1:
┌─────────┐   ┌─────────┐    ┌─────────┐   ┌─────────┐
│ Texture │ → │ Shader  │    │ Texture │ ← │ Shader  │
│   A     │   │         │    │   B     │   │         │
│ (读取)  │   │  计算   │    │ (写入)  │   │  计算   │
└─────────┘   └────┬────┘    └─────────┘   └────┬────┘
               写入 ↓                        读取 ↑
              ┌─────────┐                   ┌─────────┐
              │ Texture │                   │ Texture │
              │   B     │                   │   A     │
              │ (写入)  │                   │ (读取)  │
              └─────────┘                   └─────────┘

交换角色 ↔️  下一帧

规则:

  1. 🔵 读取: 从 Texture A 读取上一帧状态
  2. 🧮 计算: Shader 处理数据
  3. 🟢 写入: 结果写入 Texture B
  4. 🔄 交换: 下一帧 A ↔ B 互换

🔧 Three.js 实现骨架 ​

javascript
// JavaScript 端设置双缓冲
import * as THREE from 'three';

class PingPongSimulation {
    constructor(width, height) {
        // 创建两个 RenderTarget (帧缓冲)
        const options = {
            minFilter: THREE.NearestFilter,  // 精确采样
            magFilter: THREE.NearestFilter,
            format: THREE.RGBAFormat,
            type: THREE.FloatType  // 浮点纹理 (存储精确数值)
        };
        
        this.bufferA = new THREE.WebGLRenderTarget(width, height, options);
        this.bufferB = new THREE.WebGLRenderTarget(width, height, options);
        
        // 初始化材质
        this.simulationMaterial = new THREE.ShaderMaterial({
            uniforms: {
                u_buffer: { value: null },  // 上一帧状态
                u_resolution: { value: new THREE.Vector2(width, height) },
                u_dt: { value: 0.016 }  // 时间步长
            },
            fragmentShader: simulationShaderCode
        });
        
        // 用于渲染的全屏四边形
        this.scene = new THREE.Scene();
        this.camera = new THREE.OrthographicCamera(-1, 1, 1, -1, 0, 1);
        this.quad = new THREE.Mesh(
            new THREE.PlaneGeometry(2, 2),
            this.simulationMaterial
        );
        this.scene.add(this.quad);
    }
    
    update(renderer) {
        // 📖 从 A 读取
        this.simulationMaterial.uniforms.u_buffer.value = this.bufferA.texture;
        
        // ✍️ 写入到 B
        renderer.setRenderTarget(this.bufferB);
        renderer.render(this.scene, this.camera);
        
        // 🔄 交换缓冲区
        [this.bufferA, this.bufferB] = [this.bufferB, this.bufferA];
    }
    
    // 获取当前状态 (用于显示)
    getCurrentTexture() {
        return this.bufferA.texture;
    }
}

// 使用示例
const simulation = new PingPongSimulation(512, 512);

function animate() {
    requestAnimationFrame(animate);
    
    // 更新模拟
    simulation.update(renderer);
    
    // 在主屏幕显示结果
    displayMaterial.uniforms.u_simulation.value = simulation.getCurrentTexture();
    renderer.setRenderTarget(null);
    renderer.render(displayScene, camera);
}

🎮 实战案例1: Conway生命游戏 ​

📜 规则回顾 ​

Conway生命游戏是经典的细胞自动机 (Cellular Automaton):

每个格子有两种状态: 活 (1) 或 死 (0)

规则 (根据8个邻居的状态):
1️⃣ 活细胞:
   - 2-3个活邻居 → 继续存活
   - <2个活邻居 → 死亡 (孤独)
   - >3个活邻居 → 死亡 (过度拥挤)

2️⃣ 死细胞:
   - 恰好3个活邻居 → 复活 (繁殖)

可视化:

初始状态:          下一代:
□ □ □ □ □         □ □ □ □ □
□ ■ ■ ■ □   →    □ □ ■ □ □
□ □ □ □ □         □ □ ■ □ □
                  □ □ □ □ □
闪烁子 (Blinker)   旋转90°

🔧 GLSL 实现 ​

glsl
#ifdef GL_ES
precision highp float;
#endif

uniform sampler2D u_buffer;    // 上一代状态
uniform vec2 u_resolution;

void main() {
    vec2 uv = gl_FragCoord.xy / u_resolution;
    vec2 texel = 1.0 / u_resolution;  // 一个像素的UV大小
    
    // 📖 读取当前格子状态 (0 = 死, 1 = 活)
    float current = texture2D(u_buffer, uv).r;
    
    // 🔍 统计8个邻居中活细胞的数量
    float neighbors = 0.0;
    
    for(int y = -1; y <= 1; y++) {
        for(int x = -1; x <= 1; x++) {
            // 跳过中心格子自己
            if(x == 0 && y == 0) continue;
            
            vec2 offset = vec2(float(x), float(y)) * texel;
            neighbors += texture2D(u_buffer, uv + offset).r;
        }
    }
    
    // 🧮 应用规则
    float newState = 0.0;
    
    if(current > 0.5) {
        // 🟢 当前是活细胞
        if(neighbors >= 2.0 && neighbors <= 3.0) {
            newState = 1.0;  // 存活
        }
    } else {
        // ⚪ 当前是死细胞
        if(neighbors > 2.9 && neighbors < 3.1) {  // 浮点精度处理
            newState = 1.0;  // 复活
        }
    }
    
    gl_FragColor = vec4(vec3(newState), 1.0);
}

🌊 实战案例2: 水波模拟 ​

📐 物理原理 ​

水波遵循波动方程:

∂²h/∂t² = c² × ∇²h

简化为差分方程:
h(t+dt) = 2×h(t) - h(t-dt) + c²×dt² × ∇²h(t)

其中:
- h = 水面高度
- c = 波速
- ∇²h = 拉普拉斯算子 (二阶空间导数)

关键思路:

  • 需要存储两帧的状态: 当前帧 h(t) 和上一帧 h(t-dt)
  • 拉普拉斯算子 = 周围4个点的平均值 - 中心点

🔧 GLSL 实现 ​

glsl
#ifdef GL_ES
precision highp float;
#endif

uniform sampler2D u_current;  // 当前帧 h(t)
uniform sampler2D u_previous; // 上一帧 h(t-dt)
uniform vec2 u_resolution;
uniform float u_damping;      // 阻尼系数 (0.99 = 微弱衰减)

void main() {
    vec2 uv = gl_FragCoord.xy / u_resolution;
    vec2 texel = 1.0 / u_resolution;
    
    // 📖 读取当前和上一帧的高度
    float current = texture2D(u_current, uv).r;
    float previous = texture2D(u_previous, uv).r;
    
    // 🧮 计算拉普拉斯算子 (4邻域)
    float left   = texture2D(u_current, uv + vec2(-texel.x, 0.0)).r;
    float right  = texture2D(u_current, uv + vec2( texel.x, 0.0)).r;
    float top    = texture2D(u_current, uv + vec2(0.0,  texel.y)).r;
    float bottom = texture2D(u_current, uv + vec2(0.0, -texel.y)).r;
    
    float laplacian = (left + right + top + bottom) * 0.25 - current;
    
    // 🌊 波动方程 (简化版)
    float waveSpeed = 0.2;
    float newHeight = 2.0 * current - previous + laplacian * waveSpeed;
    
    // 💨 应用阻尼 (防止永久振荡)
    newHeight *= u_damping;
    
    gl_FragColor = vec4(vec3(newHeight), 1.0);
}

🎨 渲染水波 ​

glsl
uniform sampler2D u_waterHeight;
uniform vec3 u_lightDir;

void main() {
    vec2 uv = gl_FragCoord.xy / u_resolution;
    vec2 texel = 1.0 / u_resolution;
    
    // 📊 读取高度
    float height = texture2D(u_waterHeight, uv).r;
    
    // 🧭 计算法线 (通过高度梯度)
    float dx = texture2D(u_waterHeight, uv + vec2(texel.x, 0.0)).r
             - texture2D(u_waterHeight, uv - vec2(texel.x, 0.0)).r;
    float dy = texture2D(u_waterHeight, uv + vec2(0.0, texel.y)).r
             - texture2D(u_waterHeight, uv - vec2(0.0, texel.y)).r;
    
    vec3 normal = normalize(vec3(-dx, -dy, 1.0));
    
    // 💡 简单光照
    float diffuse = max(0.0, dot(normal, u_lightDir));
    
    // 🎨 水的颜色
    vec3 waterColor = vec3(0.1, 0.3, 0.6);
    vec3 color = waterColor * diffuse;
    
    // ✨ 高光
    vec3 viewDir = vec3(0.0, 0.0, 1.0);
    vec3 halfDir = normalize(u_lightDir + viewDir);
    float specular = pow(max(0.0, dot(normal, halfDir)), 32.0);
    color += specular;
    
    gl_FragColor = vec4(color, 1.0);
}

🎨 实战案例3: 反应扩散系统 (Reaction-Diffusion) ​

🧬 数学模型 ​

反应扩散是图灵斑图 (Turing Patterns) 的基础,用于模拟生物纹理:

Gray-Scott 模型:

∂A/∂t = DA × ∇²A - A×B² + f×(1-A)
∂B/∂t = DB × ∇²B + A×B² - (k+f)×B

其中:
- A, B: 两种化学物质的浓度
- DA, DB: 扩散系数
- f: 供给速率 (feed rate)
- k: 消耗速率 (kill rate)

生物意义:

  • A = "食物" (不断补充)
  • B = "细菌" (消耗A并繁殖)

🔧 GLSL 实现 ​

glsl
#ifdef GL_ES
precision highp float;
#endif

uniform sampler2D u_buffer;
uniform vec2 u_resolution;
uniform float u_feed;      // f: 0.02 ~ 0.06
uniform float u_kill;      // k: 0.045 ~ 0.065
uniform float u_dt;        // 时间步长

// 扩散系数 (常量)
const float DA = 1.0;
const float DB = 0.5;

void main() {
    vec2 uv = gl_FragCoord.xy / u_resolution;
    vec2 texel = 1.0 / u_resolution;
    
    // 📖 读取当前浓度 (r = A, g = B)
    vec2 current = texture2D(u_buffer, uv).rg;
    float A = current.r;
    float B = current.g;
    
    // 🧮 计算拉普拉斯算子 (9点模板,更精确)
    vec2 laplacian = vec2(0.0);
    
    // 中心权重
    laplacian += texture2D(u_buffer, uv).rg * -1.0;
    
    // 4邻域 (权重 0.2)
    laplacian += texture2D(u_buffer, uv + vec2(-texel.x, 0.0)).rg * 0.2;
    laplacian += texture2D(u_buffer, uv + vec2( texel.x, 0.0)).rg * 0.2;
    laplacian += texture2D(u_buffer, uv + vec2(0.0, -texel.y)).rg * 0.2;
    laplacian += texture2D(u_buffer, uv + vec2(0.0,  texel.y)).rg * 0.2;
    
    // 4对角 (权重 0.05)
    laplacian += texture2D(u_buffer, uv + vec2(-texel.x, -texel.y)).rg * 0.05;
    laplacian += texture2D(u_buffer, uv + vec2( texel.x, -texel.y)).rg * 0.05;
    laplacian += texture2D(u_buffer, uv + vec2(-texel.x,  texel.y)).rg * 0.05;
    laplacian += texture2D(u_buffer, uv + vec2( texel.x,  texel.y)).rg * 0.05;
    
    float laplaceA = laplacian.r;
    float laplaceB = laplacian.g;
    
    // 🧪 Gray-Scott 反应项
    float reaction = A * B * B;
    
    float dA = DA * laplaceA - reaction + u_feed * (1.0 - A);
    float dB = DB * laplaceB + reaction - (u_kill + u_feed) * B;
    
    // ⏱️ 欧拉积分
    A += dA * u_dt;
    B += dB * u_dt;
    
    // 🔒 确保浓度在 [0, 1] 范围
    A = clamp(A, 0.0, 1.0);
    B = clamp(B, 0.0, 1.0);
    
    gl_FragColor = vec4(A, B, 0.0, 1.0);
}

🖌️ 实战案例4: 水彩效果 ​

🎨 水彩的视觉特征 ​

真实水彩画有这些特点:

  1. 🌊 边缘累积 (Edge Darkening): 颜料在边缘聚集
  2. 💧 渗透扩散: 颜色自然晕开
  3. 🎭 纸张纹理: 底层纸张的纤维
  4. 🌈 颜色混合: 多层透明叠加

🔧 边缘累积实现 ​

原理: 颜料会向曲率高的地方流动

glsl
uniform sampler2D u_buffer;    // 当前颜料分布
uniform sampler2D u_original;  // 原始图像
uniform vec2 u_resolution;

void main() {
    vec2 uv = gl_FragCoord.xy / u_resolution;
    vec2 texel = 1.0 / u_resolution;
    
    vec4 current = texture2D(u_buffer, uv);
    
    // 🧮 计算边缘 (Sobel 算子)
    float gx = 0.0, gy = 0.0;
    
    for(int y = -1; y <= 1; y++) {
        for(int x = -1; x <= 1; x++) {
            vec2 offset = vec2(float(x), float(y)) * texel;
            float intensity = texture2D(u_original, uv + offset).r;
            
            // Sobel 核
            gx += intensity * float(x);
            gy += intensity * float(y);
        }
    }
    
    float edge = length(vec2(gx, gy));
    
    // 🌊 向边缘累积颜料
    float accumulation = edge * 0.1;
    current.rgb *= (1.0 + accumulation);
    
    gl_FragColor = current;
}

🌊 扩散模拟 ​

glsl
uniform sampler2D u_buffer;
uniform vec2 u_resolution;
uniform float u_diffusionRate;  // 扩散速率

void main() {
    vec2 uv = gl_FragCoord.xy / u_resolution;
    vec2 texel = 1.0 / u_resolution;
    
    vec4 current = texture2D(u_buffer, uv);
    
    // 🧮 4邻域平均 (扩散)
    vec4 neighbors = vec4(0.0);
    neighbors += texture2D(u_buffer, uv + vec2(-texel.x, 0.0));
    neighbors += texture2D(u_buffer, uv + vec2( texel.x, 0.0));
    neighbors += texture2D(u_buffer, uv + vec2(0.0, -texel.y));
    neighbors += texture2D(u_buffer, uv + vec2(0.0,  texel.y));
    neighbors *= 0.25;
    
    // 🌊 扩散 (当前值和邻居平均混合)
    vec4 diffused = mix(current, neighbors, u_diffusionRate);
    
    gl_FragColor = diffused;
}
