函数式编程 ​

  1. 编程规范
  • 面向过程 先做这个,再做那个，然后做什么
  • 面向对象 把功能组织成对象,然后相关操作作为对象的方法
  • 函数式编程 把功能分解为一系列独立的函数,通过函数间互相调用来完成功能
  1. 函数式编程的好处
  • 函数式编程能提高复用性和可扩展性
  • 打包优化，Thee-shaking的本质是通过文档流的引入判断是否使用某个方法，但是面向对象的编程方案无法记录
  1. 如何写好函数式编程
  • 保证纯函数
    • 一个函数的返回结果只依赖于它的参数（不依赖外部数据），同样的输入必定有同样的输出
  • 尽可能减少函数副作用
    • 函数副作用就是函数会影响外部的数据，比如全局变量
    • 如果依赖外部变量，想要不影响外部数据
      • 基本数据类型，可以考虑直接当做参数传入
      • 如果是对象/数组，可以拷贝一份再进行操作

compose和pipe函数 ​

  1. 函数式编程后执行的一些问题
  • 值的传递写起来不方便
  • 连续调用写起来很麻烦

  1. compose 函数可以接收多个独立的函数作为参数，然后将这些函数进行组合串联，最终返回一个“组合函数”。 “组合函数”执行时，其内部的所有函数都会按照组合时的顺序并以队列的形式有序的执行，前一个函数的返回值会作为下一个函数的参数被接收，因此“组合函数”中的第一个执行的函数可以接收多个参数，而之后的函数只能接收一个参数（上一个函数的返回值）。像这样一个或多个指定的参数会从“组合函数”的入口函数（第一个执行的函数）中被传入，而之后则会在多个组合串联的函数管道中进行加工、传输和输出。

  2. 实现

    • reduce实现
      js
        function compose(...funcs) {
    return function(arg) {
      return funcs.reduceRight((acc, func) => func(acc), arg);
    };
  }
    • 递归
      js
        function compose(...funcs) {
    return function(arg) {
      if (funcs.length === 0) {
        return arg;
      }
      const lastFunc = funcs.pop();
      return lastFunc(compose(...funcs)(arg));
    };
  }

高阶函数 ​

  1. 定义
  • 高阶函数是指可以接受其他函数作为参数，或者返回一个函数的函数。这种函数能够将函数作为参数传递给其他函数，或者将函数作为结果返回给调用者。
  1. 手写内置的高阶函数
  • forEach
    js
        Array.proptotype.myForEach = function(callback){
      let len = this.length
      if(typeof callback !== 'function'){
        throw new Error('callback must be function')
      }
      for(let i = 0;i < len;i++){
        callback.call(this,this[i],i)
      }
    }
  • map

        Array.proptotype.myMap = function(callback){

      }

      return resultArr
    }
  • reduce

函数柯里化 ​

  1. 定义：柯里化是一种函数的转换，它是指将一个函数从可调用的 f(a, b, c) 转换为可调用的 f(a)(b)(c)。柯里化不会调用函数。它只是对函数进行转换。

发布/订阅（社区方案） ​

  1. 发布订阅简单实现（解决回调问题）
    js
        class PubSub {
      constructor() {
        this.eventMap = {} //消息中心
      }
      publish(msg, ...rest) { //发布者
        this.eventMap[msg] && this.eventMap.forEach(cb => {
          cb(...rest)
        });
      }

      subscribe(msg, cb) { //订阅者
        this.eventMap[msg] = this.eventMap[msg] || []
        this.eventMap[msg].push(cb)
      }
    }

    //使用
    const a = new PubSub()
    a.publish('success', '任务已完成')
    setTimeout(() => {
      a.subscribe('success', (val) => console.log(`${val}!!`))
    }, 2000);

Promise ​

  • Promise并没有解决地狱回调，只是将异步控制权反转，掌握到了自己手中
  • Promise的缺点：
      1. 无法取消Promise，一旦新建它就会立即执行，无法中途取消
      1. 无法得知Promise的状态，只能通过回调函数来判断
      1. 无法得知Promise的返回值
      1. 无法捕获错误

Generator（半自动化） ​

  1. 协程 协程是一种基于线程之上，但又比线程更加轻量级的存在，这种由程序员自己写程序来管理的轻量级线程叫做“用户空间线程”，具有对内核来说不可见的特性。 正如一个进程可以拥有多个线程一样，一个线程也可以拥有多个协程

  2. 协程的Generator函数实现

    • generator函数返回的是内部指针对象，并且不会直接执行函数体
    • 当调用next方法的时候，会移动指针指向下一个阶段（阶段由yield划分）
    • next方法需要带着上一个阶段的value值作为参数，不然会是undefined
    js
        function* gen(x) {
      var y = yield x + 2  //一阶段返回结果
      return y
    }

    var g = gen(1)
    console.log(g.next()); //{value: 3, done: false}
    console.log(g.next(2)); //{value: 2, done: true}

async/await（终极解决方案，Generator函数语法糖） ​

  1. 同时触发
js
  //写法一
    let [foo, bar] = await Promise.all([getFoo(), getBar()])

  //写法二（同步执行，然后再await返回结果）
    let fooPromise = getFoo()
    let barPromise = getBar()
    let foo = await fooPromise
    let bar = await barPromise
  1. 注意不要再并发执行的函数中使用await
js
    function dbFuc(db) {
      let docs = [{}, {}, {}]
      //可能会得到错误结果,forEach的回调函数是并发执行的
      docs.forEach(async (doc) => {
        await db.add(doc)
      })
    }

    //正确做法
    function dbFuc(db) {
      let docs = [{}, {}, {}]

      for(let doc of docs){
        await db.add(doc)
      }
    }
