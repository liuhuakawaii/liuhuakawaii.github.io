性能优化 - 虚拟滚动 🚀 ​

前言 ​

在项目开发中，我遇到了一个棘手的性能问题：我们的产品是一个无限滚动加载图片展示的应用，随着用户不断滚动加载更多图片，页面中的DOM节点数量呈指数级增长。当滚动加载的图片越来越多时，页面性能急剧下降，用户体验变得糟糕。

这种情况下，虚拟滚动成为了我们的救星。那么，什么是虚拟滚动？虚拟滚动又该如何实现呢？让我们深入探讨这个前端性能优化的利器。

什么是虚拟滚动？ 💡 ​

虚拟滚动是一种针对大量数据列表的高效渲染优化技术。简单来说，它的核心思想是只渲染用户当前可见区域的DOM元素，将不在可视范围内的元素从DOM树中移除，从而大幅提升页面性能。

🎯 核心理念：眼不见，心不烦 —— 看不到的内容就不渲染

为什么需要虚拟滚动？ 🤔 ​

当页面中存在过多DOM节点时，会带来以下问题：

  • 🐌 内存占用过高：每个DOM节点都会消耗内存资源
  • ⏰ 渲染时间过长：浏览器需要计算更多元素的布局和样式
  • 🎭 页面卡顿严重：滚动、交互等操作响应缓慢
  • 📱 移动端体验差：在配置较低的移动设备上表现更为明显

性能数据对比 📊 ​

DOM节点数量内存占用初始渲染时间滚动FPS适用场景
100个~2MB~50ms60fps✅ 轻量级列表
1,000个~20MB~200ms45fps⚠️ 中等规模
10,000个~200MB~800ms15fps❌ 需要优化

虚拟滚动的实现方案对比 🔄 ​

在深入实现之前，我们需要了解虚拟滚动的两种主要实现方案，它们各有优劣，适用于不同的场景。

方案一：Padding 占位法 📐 ​

实现原理： 在容器中使用正常的文档流布局，只渲染可见部分的数据，通过padding-top和padding-bottom来撑起总高度，维持滚动条的正确长度。

核心代码示例：

html
<div class="virtual-container" style="height: 400px; overflow-y: auto;">
  <div class="virtual-list" style="
    padding-top: 1000px;    /* 前面被隐藏项目的总高度 */
    padding-bottom: 2000px; /* 后面被隐藏项目的总高度 */
  ">
    <!-- 只渲染可见的几十个项目 -->
    <div class="item">可见项目1</div>
    <div class="item">可见项目2</div>
    <div class="item">可见项目3</div>
  </div>
</div>

适用场景与优劣分析：

方面表现说明
✅ 优势
实现简单⭐⭐⭐⭐⭐不需要复杂的位置计算
多列布局⭐⭐⭐⭐⭐浏览器自动处理换行和排列
CSS兼容性⭐⭐⭐⭐⭐不依赖absolute定位
❌ 劣势
精确控制⭐⭐难以随机访问特定位置
动态高度⭐⭐需要预估或测量才能正确撑高
性能优化⭐⭐⭐padding变化可能触发重排

方案二：绝对定位法 🎯 ​

实现原理： 容器使用position: relative，每个项目通过position: absolute精确计算位置，通过top和left属性控制元素的确切位置。

核心代码示例：

html
<div class="virtual-container" style="
  height: 400px; 
  overflow-y: auto; 
  position: relative;
">
  <!-- 撑起滚动条的占位元素 -->
  <div style="height: 100000px;"></div>
  
  <!-- 绝对定位的可见项目 -->
  <div class="item" style="position: absolute; top: 0px; left: 0;">项目1</div>
  <div class="item" style="position: absolute; top: 50px; left: 0;">项目2</div>
  <div class="item" style="position: absolute; top: 100px; left: 0;">项目3</div>
</div>

适用场景与优劣分析：

方面表现说明
✅ 优势
精确控制⭐⭐⭐⭐⭐完全控制每个元素的位置
滚动性能⭐⭐⭐⭐⭐位置变化不影响其他元素
固定高度⭐⭐⭐⭐⭐计算简单，性能最优
❌ 劣势
实现复杂度⭐⭐需要手动计算top/left
多列布局⭐⭐复杂场景下计算困难
动态高度⭐需要额外的测量和缓存逻辑

🔑 技术选型指南 ​

选择Padding占位法的场景：

  • ✅ 多列网格布局（如图片墙、卡片网格）
  • ✅ 响应式布局，列数随屏幕变化
  • ✅ 项目高度相对固定且简单
  • ✅ 对位置精确控制要求不高

选择绝对定位法的场景：

  • ✅ 单列列表（如聊天记录、文章列表）
  • ✅ 固定高度的项目
  • ✅ 需要精确控制滚动位置
  • ✅ 性能要求极高的场景

从简单到复杂：循序渐进的实现 🛠️ ​

核心JavaScript实现：

class SimpleVirtualList {
  constructor(container, data, itemHeight = 50) {
    this.container = container;
    this.data = data;
    this.itemHeight = itemHeight;
    this.visibleCount = Math.ceil(container.clientHeight / itemHeight) + 2; // +2作为缓冲
    
    this.spacer = container.querySelector('#spacer');
    this.visibleItems = container.querySelector('#visible-items');
    
    this.init();
  }
  
  init() {
    // 设置总高度
    this.spacer.style.height = (this.data.length * this.itemHeight) + 'px';
    
    // 监听滚动事件
    this.container.addEventListener('scroll', () => this.handleScroll());
    
    // 初始渲染
    this.render();
  }
  
  handleScroll() {
    this.render();
  }
  
  render() {
    const scrollTop = this.container.scrollTop;
    const startIndex = Math.floor(scrollTop / this.itemHeight);
    const endIndex = Math.min(
      this.data.length - 1, 
      startIndex + this.visibleCount
    );
    
    // 清空现有项目
    this.visibleItems.innerHTML = '';
    
    // 渲染可见项目
    for (let i = startIndex; i <= endIndex; i++) {
      const item = this.createItem(this.data[i], i);
      this.visibleItems.appendChild(item);
    }
  }
  
  createItem(data, index) {
    const item = document.createElement('div');
    item.className = 'list-item';
    item.style.top = (index * this.itemHeight) + 'px';
    item.textContent = `项目 ${index}: ${data.title}`;
    return item;
  }
}

// 使用示例
const data = Array.from({ length: 10000 }, (_, i) => ({
  id: i,
  title: `数据项 ${i}`,
  content: `这是第 ${i} 个数据项的内容`
}));

const container = document.getElementById('viewport');
const virtualList = new SimpleVirtualList(container, data);

第二步：支持动态高度的复杂实现 ​

当项目高度不固定时，我们需要实现测量和缓存机制。

动态高度虚拟列表实现：

js
class DynamicHeightVirtualList {
  constructor(container, data) {
    this.container = container;
    this.data = data;
    this.estimatedItemHeight = 80; // 预估高度
    this.heights = new Array(data.length).fill(this.estimatedItemHeight);
    this.positions = [0]; // 前缀和数组，存储每个位置的累计高度
    
    this.spacer = container.querySelector('#spacer');
    this.visibleItems = container.querySelector('#visible-items');
    this.renderedItems = new Map(); // 缓存已渲染的DOM元素
    
    this.init();
  }
  
  init() {
    this.updatePositions();
    this.container.addEventListener('scroll', () => this.handleScroll());
    this.render();
  }
  
  // 更新位置缓存（前缀和）
  updatePositions() {
    for (let i = 1; i <= this.data.length; i++) {
      this.positions[i] = this.positions[i - 1] + this.heights[i - 1];
    }
    this.spacer.style.height = this.positions[this.data.length] + 'px';
  }
  
  // 二分查找：根据scrollTop找到起始索引
  findStartIndex(scrollTop) {
    let left = 0, right = this.positions.length - 1;
    
    while (left < right) {
      const mid = Math.floor((left + right) / 2);
      if (this.positions[mid] < scrollTop) {
        left = mid + 1;
      } else {
        right = mid;
      }
    }
    
    return Math.max(0, left - 1);
  }
  
  handleScroll() {
    this.render();
  }
  
  render() {
    const scrollTop = this.container.scrollTop;
    const containerHeight = this.container.clientHeight;
    
    // 计算可见范围
    const startIndex = this.findStartIndex(scrollTop);
    const endIndex = this.findEndIndex(scrollTop + containerHeight);
    
    // 移除不可见的项目
    for (const [index, item] of this.renderedItems) {
      if (index < startIndex || index > endIndex) {
        this.visibleItems.removeChild(item);
        this.renderedItems.delete(index);
      }
    }
    
    // 添加新的可见项目
    for (let i = startIndex; i <= endIndex; i++) {
      if (!this.renderedItems.has(i)) {
        const item = this.createItem(this.data[i], i);
        this.visibleItems.appendChild(item);
        this.renderedItems.set(i, item);
        
        // 测量实际高度
        this.measureItem(item, i);
      }
    }
  }
  
  findEndIndex(bottom) {
    let index = this.findStartIndex(bottom);
    while (index < this.data.length && this.positions[index] < bottom) {
      index++;
    }
    return Math.min(index, this.data.length - 1);
  }
  
  measureItem(item, index) {
    // 使用 ResizeObserver 或定时器测量
    const height = item.offsetHeight;
    if (height !== this.heights[index]) {
      const delta = height - this.heights[index];
      this.heights[index] = height;
      
      // 更新后续位置
      for (let i = index + 1; i <= this.data.length; i++) {
        this.positions[i] += delta;
      }
      
      this.spacer.style.height = this.positions[this.data.length] + 'px';
    }
  }
  
  createItem(data, index) {
    const item = document.createElement('div');
    item.className = 'list-item dynamic-height';
    item.style.position = 'absolute';
    item.style.top = this.positions[index] + 'px';
    item.style.width = '100%;
    
    // 模拟不同高度的内容
    item.innerHTML = `
      <h3>项目 ${index}</h3>
      <p>${data.content}</p>
      ${data.hasImage ? '<div style="height: 100px; background: #f0f0f0;">图片区域</div>' : ''}
    `;
    
    return item;
  }
}

第三步：无限滚动场景的内存管理 ​

对于分页加载的无限滚动场景，我们需要实现智能的内存管理。

内存优化的虚拟列表：

js
class InfiniteVirtualList extends DynamicHeightVirtualList {
  constructor(container, loadMoreCallback, maxItems = 1000) {
    super(container, []);
    this.loadMoreCallback = loadMoreCallback;
    this.maxItems = maxItems; // 最大保留项目数
    this.loading = false;
    this.hasMore = true;
    this.topOffset = 0; // 顶部偏移，用于内存管理
    
    this.loadInitialData();
  }
  
  async loadInitialData() {
    const data = await this.loadMoreCallback(0);
    this.data = data;
    this.heights = new Array(data.length).fill(this.estimatedItemHeight);
    this.positions = [0];
    this.updatePositions();
    this.render();
  }
  
  async handleScroll() {
    super.handleScroll();
    
    // 检查是否需要加载更多数据
    const scrollTop = this.container.scrollTop;
    const scrollHeight = this.container.scrollHeight;
    const clientHeight = this.container.clientHeight;
    
    if (!this.loading && this.hasMore && 
        scrollTop + clientHeight > scrollHeight - 1000) {
      await this.loadMore();
    }
    
    // 检查是否需要清理早期数据
    if (this.data.length > this.maxItems) {
      this.pruneEarlyData();
    }
  }
  
  async loadMore() {
    this.loading = true;
    
    try {
      const newData = await this.loadMoreCallback(this.data.length);
      if (newData.length === 0) {
        this.hasMore = false;
        return;
      }
      
      // 添加新数据
      this.data.push(...newData);
      
      // 扩展高度数组
      const newHeights = new Array(newData.length).fill(this.estimatedItemHeight);
      this.heights.push(...newHeights);
      
      this.updatePositions();
      this.render();
      
    } finally {
      this.loading = false;
    }
  }
  
  // 清理早期数据，保持内存使用合理
  pruneEarlyData() {
    const removeCount = this.data.length - this.maxItems;
    if (removeCount <= 0) return;
    
    // 计算被移除部分的高度
    const removedHeight = this.positions[removeCount];
    
    // 移除数据
    this.data.splice(0, removeCount);
    this.heights.splice(0, removeCount);
    
    // 重新计算位置
    const newPositions = [0];
    for (let i = 1; i <= this.data.length; i++) {
      newPositions[i] = newPositions[i - 1] + this.heights[i - 1];
    }
    this.positions = newPositions;
    
    // 调整滚动位置和偏移
    this.topOffset += removedHeight;
    this.container.scrollTop -= removedHeight;
    
    // 清理渲染缓存
    this.renderedItems.clear();
    this.visibleItems.innerHTML = '';
    
    this.updatePositions();
    this.render();
  }
  
  updatePositions() {
    super.updatePositions();
    // 添加顶部偏移
    this.spacer.style.height = (this.topOffset + this.positions[this.data.length]) + 'px';
  }
  
  createItem(data, index) {
    const item = super.createItem(data, index);
    // 调整位置，加上顶部偏移
    item.style.top = (this.topOffset + this.positions[index]) + 'px';
    return item;
  }
}

// 使用示例
const container = document.getElementById('viewport');
const virtualList = new InfiniteVirtualList(
  container,
  async (offset) => {
    // 模拟API调用
    const response = await fetch(`/api/data?offset=${offset}&limit=50`);
    return response.json();
  },
  1000 // 最多保留1000项数据在内存中
);

class DOMPool {
  constructor(createFn) {
    this.createFn = createFn;
    this.pool = [];
  }

  }

function useVirtualList(items, itemHeight, containerHeight) {
  const [scrollTop, setScrollTop] = useState(0);
  
  const visibleRange = useMemo(() => {
    const startIndex = Math.floor(scrollTop / itemHeight);
    const endIndex = Math.min(
      items.length - 1,
      startIndex + Math.ceil(containerHeight / itemHeight) + 1
    );
    return { startIndex, endIndex };
  }, [scrollTop, itemHeight, containerHeight, items.length]);
  
  const visibleItems = useMemo(() => 
    items.slice(visibleRange.startIndex, visibleRange.endIndex + 1)
  , [items, visibleRange]);
  
  return {
    visibleItems,
    scrollTop,
    setScrollTop,
    totalHeight: items.length * itemHeight,
    offsetY: visibleRange.startIndex * itemHeight
  };
}

实战案例：响应式网格的虚拟滚动实现 🎯 ​

在实际项目中，我们经常需要处理更复杂的场景：响应式多列网格布局 + 无限滚动 + 虚拟化优化。下面分享一个真实项目中的完整实现案例。

项目背景 ​

这是一个图片/卡片展示应用，需要支持：

  • 📱 响应式布局：不同屏幕尺寸显示不同列数（2-6列）
  • ♾️ 无限滚动：分页加载大量数据
  • ⚡ 虚拟滚动：优化渲染性能，支持数万张卡片
  • 🎨 动态高度：卡片高度根据屏幕宽度自适应

核心实现思路 ​

与前面介绍的单列虚拟滚动不同，这个案例使用了Padding占位法 + 多列网格的方案，更适合响应式布局场景。

🔑 关键设计决策：

  • 选择Padding占位法而非绝对定位，因为多列布局的位置计算极其复杂
  • 使用行级虚拟化而非单项虚拟化，简化计算逻辑
  • 响应式配置统一管理，避免重复计算

完整实现代码 ​

jsx
// React Hook实现：响应式网格虚拟滚动
function useResponsiveVirtualGrid(cards) {
    const virtualizedContainerRef = useRef(null);
    const { isAbove } = useBreakpoint();

    // 虚拟化配置常量
    const VIRTUALIZATION_CONFIG = {
        gap: 16, // 行间距
        bufferRows: 5, // 缓冲行数，避免滚动时闪烁
    };

    // 响应式布局配置 - 统一管理所有断点相关的配置
    const getLayoutConfig = useCallback(() => {
        const vw = window.innerWidth;

        switch (true) {
            case isAbove('3xl'):
                return {
                    columns: 6,
                    cardHeight: 341,
                    renderRows: 7,
                };
            case isAbove('2xl'):
                return {
                    columns: 6,
                    cardHeight: 284,
                    renderRows: 6,
                };
            case isAbove('xl'):
                return {
                    columns: 5,
                    cardHeight: Math.max(180, (0.9 * vw - 346) / 5 - 28),
                    renderRows: 5,
                };
            case isAbove('xmd'):
                return {
                    columns: 4,
                    cardHeight: Math.max(180, (0.9 * vw - 316) / 4 - 16),
                    renderRows: 4,
                };
            case isAbove('sm'):
                return {
                    columns: 3,
                    cardHeight: Math.max(180, (0.9 * vw - 316) / 3 - 16),
                    renderRows: 3,
                };
            default:
                return {
                    columns: 2,
                    cardHeight: 180,
                    renderRows: 3,
                };
        }
    }, [isAbove]);

    // 虚拟化状态
    const [virtualizedState, setVirtualizedState] = useState({
        startIndex: 0,
        endIndex: 0,
        paddingTop: 0,
        paddingBottom: 0,
        visibleItems: [],
    });

    // 核心虚拟化计算函数
    const calculateVirtualization = useCallback(() => {
        const container = virtualizedContainerRef.current;
        if (!container || cards.length === 0) {
            return {
                startIndex: 0,
                endIndex: 0,
                paddingTop: 0,
                paddingBottom: 0,
                visibleItems: [],
            };
        }

        const layoutConfig = getLayoutConfig();
        const { columns, cardHeight, renderRows } = layoutConfig;
        const { gap, bufferRows } = VIRTUALIZATION_CONFIG;

        const rowHeight = cardHeight + gap;
        const totalRows = Math.ceil(cards.length / columns);
        const containerRect = container.getBoundingClientRect();
        const viewportHeight = window.innerHeight;

        // 计算可见区域
        const scrollTop = Math.max(0, -containerRect.top);
        const scrollBottom = scrollTop + viewportHeight;

        // 计算可见行范围
        const visibleStartRow = Math.floor(scrollTop / rowHeight);
        const visibleEndRow = Math.min(
            totalRows,
            Math.ceil(scrollBottom / rowHeight)
        );

        // 添加缓冲区
        const renderStartRow = Math.max(0, visibleStartRow - bufferRows);
        const renderEndRow = Math.min(totalRows, visibleEndRow + bufferRows);

        // 计算渲染索引范围
        const startIndex = renderStartRow * columns;
        const endIndex = Math.min(cards.length - 1, renderEndRow * columns - 1);

        // 计算 padding
        const paddingTop = renderStartRow * rowHeight;
        const paddingBottom = Math.max(
            0,
            (totalRows - renderEndRow) * rowHeight
        );

        // 获取可见的卡片数据
        const visibleItems = cards.slice(startIndex, endIndex + 1);

        return {
            startIndex,
            endIndex,
            paddingTop,
            paddingBottom,
            visibleItems,
            // 调试信息
            debug: {
                totalRows,
                rowHeight,
                visibleStartRow,
                visibleEndRow,
                renderStartRow,
                renderEndRow,
                scrollTop,
                scrollBottom,
                columns,
                cardHeight,
            },
        };
    }, [cards, getLayoutConfig]);

    // 更新虚拟化状态
    const updateVirtualization = useCallback(() => {
        const newState = calculateVirtualization();
        setVirtualizedState(newState);

        // 开发环境下的调试日志
        if (process.env.NODE_ENV === 'development') {
            console.log('虚拟化更新:', {
                startIndex: newState.startIndex,
                endIndex: newState.endIndex,
                visibleCount: newState.visibleItems.length,
                paddingTop: newState.paddingTop,
                paddingBottom: newState.paddingBottom,
                debug: newState.debug,
            });
        }
    }, [calculateVirtualization]);

    // 节流的滚动处理
    const throttledUpdateVirtualization = useCallback(
        useThrottle(updateVirtualization, 16), // 约60fps
        [updateVirtualization]
    );

    useEffect(() => {
        const handleScroll = () => throttledUpdateVirtualization();
        const handleResize = () => updateVirtualization(); // resize不需要节流

        window.addEventListener('scroll', handleScroll, { passive: true });
        window.addEventListener('resize', handleResize);

        // 初始化
        updateVirtualization();

        return () => {
            window.removeEventListener('scroll', handleScroll);
            window.removeEventListener('resize', handleResize);
        };
    }, [throttledUpdateVirtualization, updateVirtualization]);

    useEffect(() => {
        updateVirtualization();
    }, [cards.length, updateVirtualization]);

            // 返回虚拟化相关数据和引用
    return {
        virtualizedContainerRef,
        virtualizedState,
        layoutConfig: getLayoutConfig()
    };
}

// 组件中的使用示例
function GalleryComponent({ cards, loadMore, allLoaded, isLoading }) {
    const { 
        virtualizedContainerRef, 
        virtualizedState,
        layoutConfig 
    } = useResponsiveVirtualGrid(cards);

    return (
        <div
            className={`gallery-container h-full pb-[40px] ${
                cards.length === 0 ? 'overflow-hidden' : ''
            }`}
            ref={virtualizedContainerRef}
            style={{
                paddingTop: virtualizedState.paddingTop,
                paddingBottom: virtualizedState.paddingBottom,
            }}
        >
            <ScrollLoader
                allLoaded={allLoaded}
                loadMore={loadMore}
                width="100%"
            >
                {isLoading ? (
                    <LoadingSkeleton />
                ) : cards.length === 0 ? (
                    <EmptyComponent />
                ) : (
                    virtualizedState.visibleItems.map((card, index) => {
                        const actualIndex = virtualizedState.startIndex + index;
                        return (
                            <CardComponent
                                key={card.task_uuid || actualIndex}
                                card={card}
                                index={actualIndex}
                                layoutConfig={layoutConfig}
                            />
                        );
                    })
                )}
            </ScrollLoader>
        </div>
    );
}

实战经验总结 💡 ​

通过这个真实项目案例，我们可以总结出以下实战经验：

  1. 🎯 方案选择要务实：

    • 理论上绝对定位性能最优，但多列响应式场景下Padding占位法更实用
    • 不要为了性能而牺牲代码的可维护性

    • 现代Web应用必须支持多设备适配

  3. ⚡ 性能优化要适度：

    • 60fps的流畅度已经足够，过度优化反而增加复杂性
    • 节流、缓冲区等策略比极致的算法优化更有实用价值

  4. 🐛 可观测性很重要：

    • 开发环境的调试信息对排查问题至关重要
    • 生产环境要确保调试代码被完全移除
