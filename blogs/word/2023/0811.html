setInterval 工作随记与最佳实践 ⚠️ ​

基本概念 📝 ​

setInterval 是 JavaScript 中用于设置循环定时器的函数，它可以按照指定的时间间隔重复执行某个任务。

常见陷阱和注意事项 🚨 ​

1. 首次执行时机 ​

  • ⏰ setInterval 在创建后不会立即执行回调函数
  • 🔄 而是等待第一个时间间隔后才开始执行
  • 💡 如果需要立即执行，可以先调用一次函数，再设置 interval
js
// 不推荐
setInterval(callback, 1000);
// 推荐：需要立即执行时
callback(); // 立即执行一次
setInterval(callback, 1000);

2. 闭包陷阱 ​

  • 📦 setInterval 创建的是一个闭包
  • 🔒 回调函数会保持对创建时变量的引用
  • ⚠️ 在定时器执行过程中修改外部变量，不会影响到定时器内的值
js
// 错误示例
let count = 0;
const timer = setInterval(() => {
  console.log(count);
  if(count >= 5) clearInterval(timer);
  count++;
}, 1000);

// 正确示例
const timer = setInterval(() => {
  static let count = 0;
  console.log(count);
  if(count >= 5) clearInterval(timer);
  count++;
}, 1000);

3. 最佳实践 ✨ ​

  1. 始终保存定时器ID并在适当时机清除：
javascript
const timerId = setInterval(() => {
    // 你的代码
}, 1000);

// 在组件卸载或不需要时清除
clearInterval(timerId);
  1. 使用 setTimeout 替代方案：
javascript
function betterInterval(callback, delay) {
    function repeater() {
        callback();
        setTimeout(repeater, delay);
    }
    setTimeout(repeater, delay);
}

总结 📌 ​

  • 🎯 使用 setInterval 时要注意首次执行时机
  • 🔍 注意闭包特性，避免使用外部变量作为终止条件
  • 🧹 及时清理定时器，避免内存泄漏
  • 🔄 考虑使用 setTimeout 实现更灵活的循环定时方案
