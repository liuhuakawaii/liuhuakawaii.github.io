初探Chrome Extension开发 ​

参考大佬文章

在地址栏输入 chrome://extensions 访问,勾选开发者模式即可以文件夹的形式直接加载插件，否则只能安装.crx格式的文件。

插件都必须有一个配置文件（manifest.json），用于描述插件的重要信息。插件可以包含不同的模块（页面或脚本），如下图所示。

核心配置 ​

manifest.json ​

其中manifest.json是配置文件，它描述了插件的功能，比如插件名称，版本，图标，入口文件等。

json
{
  "name": "Test",  // 插件名称
  "version": "1.0",// 插件版本
  "manifest_version": 3, // 固定值，manifest 版本号

  
  "description": "chrome 插件",
  "icons": {
    "16": "icon.png", // 插件页面的 favicon
    "48": "icon.png", // 插件管理页展示的图标
    "128": "icon.png" // Chrome 商店中展示的图标
  },


  "default_locale": "en", // 默认国际化语言，若插件包含 _locales 目录，必须设定默认值
  "background": { "service_worker": "background.js"}, // 会一直常驻的后台JS或后台页面
  "permissions": [  // 允许插件访问的 url 和 API
    "contextMenus", // 右键菜单
    "tabs", // 标签
    "notifications", // 通知
    "webRequest", // web请求
    "webRequestBlocking",
    "storage", // 插件本地存储
  ],
  "options_page": "options.html", // option 页面
  "action": {
    "default_popup": "popup.html", // 点击图标展示的 popup 页面
    "default_icon": "icon.png", // 插件在浏览器工具栏中展示的图标
    "default_title": "Change page color", // hover 图标展示的文字
  },
  "host_permissions": ["https://*.example.com/"],
  "content_scripts": [ 
    {
      "matches": ["file:///*/chrome-extension-demo/detail.html*"], // 指定可以注入脚本的页面
      "js": ["common.js", "jquery.min.js", "getDetail.js"], // 注入 web 页面的 content script 脚本
       // 代码注入的时间，可选值： "document_start", "document_end", or "document_idle"，最后一个表示页面空闲时，默认document_idle
       "run_at": "document_start"
    },
    {
      "matches": ["file:///*/chrome-extension-demo/list.html"], 
      "js": ["common.js", "getList.js"]
    }
  ]
}

上述json文件中有几个属性需要重点关注：

  1. background.service_worker：background.js的入口文件，它负责插件的初始化，注册事件监听等。
  2. content_scripts：content_scripts是插件的注入文件，它负责插件的注入，注入的js文件会注入到匹配的页面中。
  3. permissions：插件需要的权限，如读取本地存储，读取tab等。
  4. action：插件的入口页面，它负责插件的显示，点击图标会跳转到这个页面。
    • 以前的版本这里分为browser_action和page_action
    • 现在可以配置host_permissions，用来指定插件可以访问的域名
  5. manifest_version：插件的版本，目前只支持3。

Background Script ​

Background Script 是在扩展程序的整个生命周期中一直运行的脚本。它通常用于处理长期任务、存储数据、管理扩展的状态等。在 Manifest V3 中，background.js 通常是一个 Service Worker。

特点

  • **持久运行：**即使没有用户交互，也一直保持活跃。
  • **全局访问权限：**可以访问大多数扩展API，包括 chrome.tabs, chrome.runtime, chrome.storage 等。
  • **独立于页面：**不直接与任何网页关联，可以独立于用户当前打开的页面执行任务。
  • **无限制跨域：**也就是可以跨域访问任何网站而无需要求对方设置CORS。

如何配置代码

json
{
  "manifest_version": 3,
  "name": "我的扩展",
  "version": "1.0",
  "background": {
    "service_worker": "background.js"
  }
}

应用场景

  • **定时任务：**例如，定期检查服务器上的更新。
  • **跨页面状态管理：**例如，维护用户的登录状态。
  • **消息传递：**作为 content_scripts 和其他扩展组件之间的中介

如何调试代码 ：插件管理页点击背景页即可

popup ​

popup是点击action图标时打开的一个小窗口网页，焦点离开网页就立即关闭，一般用来做一些临时性的交互。

popup可以包含任意你想要的 HTML 内容，并且会自适应大小（限制宽度为800px，高度600px）。可以通过default_popup字段来指定 popup 页面，也可以调用setPopup()方法。

如何调试代码 ： popup 页面右键审查元素

配置方式：

json
{
  "action":
  {
    "default_icon": "img/icon.png",
    "default_title": "这是一个示例Chrome插件",// 图标悬停时的标题，可选
    "default_popup": "popup.html"
  }
}

popup 中可以做的事情

  1. 发送消息到 background script 或 content scripts。
js
//popup.js
document.addEventListener('DOMContentLoaded', function() {
  // 初始化时加载天气信息
  loadWeather();
  // 添加刷新按钮点击事件
  document.getElementById('refreshWeather').addEventListener('click', loadWeather);

  function loadWeather() {
    // 发送消息到 background script 获取天气信息
    chrome.runtime.sendMessage({ action: 'getWeather' }, function(response) {
      if (chrome.runtime.lastError) {
        console.error(chrome.runtime.lastError.message);
        return;
      }
      document.getElementById('weatherInfo').innerText = response.weather;
    });
  }
});

//background.js
chrome.runtime.onMessage.addListener(function(request, sender, sendResponse) {
  if (request.action === 'getWeather') {
    // 模拟从服务器获取天气信息
    setTimeout(() => {
      sendResponse({ weather: '晴朗' });
    }, 1000);
  }
});
  1. 调用 chrome.storage 存储和读取数据。
js
document.addEventListener('DOMContentLoaded', function() {
  loadLoginStatus();

  document.getElementById('logout').addEventListener('click', logout);

  function loadLoginStatus() {
    // 从 storage 中获取登录状态
    chrome.storage.local.get(['isLoggedIn'], function(result) {
      if (chrome.runtime.lastError) {
        console.error(chrome.runtime.lastError.message);
        return;
      }
      document.getElementById('loginStatus').innerText = result.isLoggedIn ? '已登录' : '未登录';
    });
  }

  function logout() {
    // 将登录状态设为 false
    chrome.storage.local.set({ isLoggedIn: false }, function() {
      if (chrome.runtime.lastError) {
        console.error(chrome.runtime.lastError.message);
        return;
      }
      loadLoginStatus(); // 重新加载登录状态
    });
  }
});
  1. 调用 chrome.notifications 创建通知
js
document.addEventListener('DOMContentLoaded', function() {
  document.getElementById('createNotification').addEventListener('click', createNotification);
  function createNotification() {
    // 创建一个基本的通知
    chrome.notifications.create({
      type: 'basic',
      iconUrl: 'icons/icon128.png',
      title: '这是一个标题',
      message: '您刚才点击了创建通知按钮！'
    });
  }
});
  1. 调用 chrome.tabs 创建新标签页。
js
document.addEventListener('DOMContentLoaded', function() {
  document.getElementById('createTab').addEventListener('click', createTab);
  function createTab() {
    // 创建一个新的标签页并导航到指定 URL
    chrome.tabs.create({ url: 'https://www.example.com' });
  }
});

Content script ​

Content script 是在特定的网页上下文中运行的脚本。它可以访问该页面的 DOM，并与页面进行交互。

如何配置

json
{
  "manifest_version": 3,
  "name": "我的扩展",
  "version": "1.0",
  "content_scripts": [
    {
      "matches": ["*://*.example.com/*"],
      "js": ["content_script.js"]
    }
  ]
}

应用场景

  • **页面增强：**例如，添加额外的按钮或链接到网页。
  • **数据收集：**例如，收集用户在网页上的行为数据。
  • **功能增强：**例如，为网页添加新的功能，如翻译、注释等。

其余常用功能 ​

Badge（徽标） ​

在最新的 Chrome 浏览器扩展中，调用 Badge（徽标）的功能主要是通过 chrome.action 或 chrome.browserAction（在 Manifest V2 中）。

js
// background.js
chrome.runtime.onInstalled.addListener(function(details) {
  if (details.reason === 'install') { // 安装时设置 Badge 文本
    chrome.action.setBadgeText({ text: 'New' });
    chrome.action.setBadgeBackgroundColor({ color: [255, 0, 0, 255] });
  }
});

chrome.runtime.onMessage.addListener(function(message, sender, sendResponse) {
  if (message.type === 'set-badge') {
    // 接收到消息时设置 Badge 文本
    chrome.action.setBadgeText({ text: message.text });
    chrome.action.setBadgeBackgroundColor({ color: message.color });
  }
  return true;
});

//popup.js
document.getElementById('set-badge').addEventListener('click', function() {
  // 发送消息给 background script 设置 Badge
  chrome.runtime.sendMessage({ type: 'set-badge', text: '10', color: [255, 0, 0, 255] });
});

使用场景:

  1. 显示插件状态，如未登录、未安装等。
  2. 显示未读消息数、待办事项。
  3. 显示更新提醒。

右键菜单 ​

基本步骤

  1. 注册菜单项：使用 chrome.contextMenus.create() 方法来创建菜单项。
  2. 监听点击事件：使用 chrome.contextMenus.onClicked 事件来处理菜单项被点击时的操作。 完整contextMenus API创建右键菜单项
js
// 创建一个右键菜单项
function createMenu() {
  chrome.contextMenus.create({
    id: 'myMenu',
    title: '这是我的右键菜单：%s', // %s表示选中的文字
    contexts: ['selection'], // 上下文环境 仅当用户选中文本时显示此菜单项
    parentId: 1, // 右键菜单项的父菜单项ID。指定父菜单项将会使此菜单项成为父菜单项的子菜单
    documentUrlPatterns: 'https://*.baidu.com/*', // 只在某些页面显示此右键菜单
    onclick: function(params) {
    // 注意不能使用location.href，因为location是属于background的window对象
    chrome.tabs.create({url: 'https://www.baidu.com/s?ie=utf-8&wd=' + encodeURI(params.selectionText)});
  }
  });
}
chrome.runtime.onInstalled.addListener(createMenu); // 当扩展安装或更新时创建菜单项

chrome.contextMenus.remove(menuItemId)；// 删除某一个菜单项
chrome.contextMenus.removeAll();// 删除所有自定义右键菜单
chrome.contextMenus.update(menuItemId, updateProperties);// 更新某一个菜单项

DevTools ​

DevTools可以让你：

  • 自定义一个和多个和Elements、Console、Sources等同级别的面板；
  • 自定义侧边栏(sidebar)，目前只能自定义Elements面板的侧边栏；
  • 例如显示 DOM 信息、监控性能、记录网络请求等。这些功能可以极大地增强开发者的工具集，提高开发效率。

基本步骤

  1. 配置 manifest.json
json
{
  "manifest_version": 3,
  "name": "我的 DevTools 扩展",
  "version": "1.0",
  "description": "一个简单的 DevTools 扩展示例",
  "devtools_page": "devtools.html",  // 只能指向一个HTML文件，不能是JS文件
  "permissions": ["devtools_page"]
}
  1. 创建 DevTools 页面
js
//devtools.html ...
//devtools.js
document.addEventListener('DOMContentLoaded', function() {
  const contentDiv = document.getElementById('content');

  // 获取当前正在检查的标签页
  chrome.devtools.inspectedWindow.eval(
    'document.title',
    function(result, isException) {
      if (isException) {
        contentDiv.textContent = 'Error evaluating expression: ' + isException.message;
      } else {
        contentDiv.textContent = '当前页面的标题是: ' + result;
      }
    }
  );

  // 创建自定义侧边栏
chrome.devtools.panels.elements.createSidebarPane("Images", function (sidebar) {
  // sidebar.setPage('../sidebar.html'); // 指定加载某个页面
  sidebar.setExpression('document.querySelectorAll("img")', "All Images"); // 通过表达式来指定
  //sidebar.setObject({aaa: 111, bbb: 'Hello World!'}); // 直接设置显示某个对象
});
});

options 页面是指用户可以用来配置扩展设置的页面 这个纯靠自己发挥了，就是写一个html页面和一个js文件，然后实现功能

json

长连接和短连接 ​

Chrome 插件中有 2 种通信方式

  1. 长连接：使用 chrome.runtime.connect() 方法，或者 chrome.tabs.connect() 方法，可以保持连接，直到 disconnect() 被调用。
  2. 短连接：使用 chrome.runtime.sendMessage() 方法，或者 chrome.tabs.sendMessage() 方法，每次调用都会创建一个新的连接，然后立即断开连接。

长连接示例

js
//popup.js
getCurrentTabId((tabId) => {
  var port = chrome.tabs.connect(tabId, { name: "test-connect" });
  port.postMessage({ question: "你是谁啊？" });
  port.onMessage.addListener(function (msg) {
    alert("收到消息：" + msg.answer);
    if (msg.answer && msg.answer.startsWith("我是")) {
      port.postMessage({ question: "哦，原来是你啊！" });
    }
  });
});

//content-script.js：
// 监听长连接
chrome.runtime.onConnect.addListener(function (port) {
  console.log(port);
  if (port.name == "test-connect") {
    port.onMessage.addListener(function (msg) {
      console.log("收到长连接消息：", msg);
      if (msg.question == "你是谁啊？") port.postMessage({ answer: "我是你爸！" });
    });
  }
});
