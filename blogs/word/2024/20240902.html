初探微前端 ​

微前端（Micro Frontend）是一种架构模式，它允许将一个大型的单页面应用程序（SPA）拆分成多个独立部署的小型前端应用。这些小型应用可以在不同的团队间开发、维护，并且可以独立地部署和升级。微前端架构能够带来多方面的优势，其中包括性能优化。

微前端的优势 ​

  1. **可维护性：**将一个大的应用分解为多个小的、可管理的部分，使得每个部分都可以由不同的团队负责，提高了项目的可维护性。
  2. **可扩展性：**随着应用的增长，可以轻松地添加新的微前端应用，而不必担心对现有系统造成影响。
  3. **技术栈灵活性：**不同的微前端应用可以采用不同的技术和框架，使得团队可以根据需求选择最适合的技术栈。
  4. **独立部署：**每个微前端应用可以独立部署，无需等待整个大应用的重新构建和部署，加快了迭代速度。
  5. **性能优化：**微前端架构可以通过以下几个方面来优化项目性能：
    • **按需加载：**微前端应用可以根据需要动态加载，这意味着用户只加载和执行他们当前需要的功能，减少了初始加载时间和资源消耗。
    • **减小包体积：**每个微前端应用只需要打包和加载其自身所需的部分，而不是整个大应用的所有代码和资源，从而减小了包体积。
    • **缓存策略：**由于每个微前端应用是独立的，可以针对不同的应用制定更精细的缓存策略，提高缓存命中率。
    • **异步加载：**通过懒加载（Lazy Loading）等技术，可以在用户真正需要的时候再加载相关的微前端应用，进一步提升性能。

微前端的实现方式 ​

  1. iframe：

    • 使用iframe标签将微前端应用嵌入到主应用中，通过iframe的src属性指定微前端应用的地址，实现微前端的加载。
    • 需要通过postMessage进行通信。
    • 完美的沙箱机制自带应用隔离
    • 缺点：
      • iframe无法直接访问主应用的DOM
      • 双滚动条、弹框只能在iframe中，在内部切换刷新就会丢失状态..

  2. single-spa

    • single-spa通过路由劫持实现应用的加载（采用SystemJS），提供应用间公共组件加载及公共业务逻辑处理。子应用需要暴露固定的钩子bootstrap、mount、unmount 接入协议
    • 基于props主子应用间通信，子应用通过props.onGlobalStateChange监听全局状态变化，通过props.setGlobalState设置全局状态。
    • 缺点：
      • 配置复杂，需要自己实现各种钩子，如：bootstrap、mount、unmount、update、unmount。
      • 无沙箱机制，需要实现自己实现JS沙箱以及CSS沙箱
      • 需要对原有的应用进行改造、子应用间相同资源重复加载问题

  3. Module Federation

    • 通过模块联邦将组件进行打包导出使用
    • 共享模块的方式进行通信
    • 无CSS沙箱和JS沙箱
    • 缺点：需要webpack5

systemjs实现 ​

  1. systemjs是如何定义的，先看打包后的结果,System.register（依赖列表，callback返回一个setters、execute）
  2. react、react-dom这种如果版本一致，可以提成公共模块，在生产环境打包的时候externals配置，减少重复打包。这样可以通过cdn引入，加载后调用setters，将对应的结果赋予给webpack
  3. 本质就是先加载依赖列表，再去加载真正的逻辑，内部通过script脚本加载资源（JSONP），通过window快照来保存先后状态

<script>
    const newMapUrl = {}
    function processScript(){
        Array.from(document.querySelectorAll('script')).forEach(script=>{
            if(script.type === 'systemjs-importmap'){
                const imports = JSON.parse(script.innerHTML).imports
                Object.keys(imports).forEach(key=> newMapUrl[key] = imports[key])
            }
        })
    }

    function load(id){
        return new Promise((resolve, reject)=>{
            const script = document.createElement('script')
            script.src = newMapUrl[id] || id
            script.async = true
            document.body.appendChild(script)
            script.addEventListener('load', ()=>{
                let _lastRegister = lastRegister
                lastRegister = null
                resolve(_lastRegister)
            })
        })
    }
    let set = new Set()  //先保存window上的属性
    function saveGlobalProperty(){
        for(let k in window){
            set.add(k)
        }
    }
    saveGlobalProperty()

    function getLastGlobalProperty(){ //快照对比得到新增的属性
        for(let k in window){
            if(set.has(k)) continue 
            set.add(k) //保存新增的属性
            return window[k]
        }
    }

    let lastRegister
    class SystemJS {
        import(id){//这个id可以是一个第三方的路径cdn
            return Promise.resolve(processScript()).then(()=>{
                const lastSepIndex = location.href.lastIndexOf('/') 
                const baseUrl = location.href.slice(0, lastSepIndex + 1)
                if(id.startsWith('./')){
                    return baseUrl + id.slice(2)
                }
            }).then((id)=>{
                let execute
                return load(id).then((register)=>{
                    let { setters, exe } = register[1](()=>{})
                    execute = exe
                    // execute 是真正执行的渲染逻辑
                    // setters 是用来保存加载后的资源，加载资源调用setters
                    return [register[0],setters]
                }).then(([registeration, setters])=>{
                    return Promise.all(registeration.map((dep,i)=>{
                        return load(dep).then(res=>{
                                setters[i](getLastGlobalProperty())
                            })
                        }))   
                    }).then(()=>{
                        return execute()
                    })
            })
        }

        register(deps, declare){
            lastRegister = [deps, declare]
        }
    }
    const System = new SystemJS()
    System.import('./app.js').then(res=>{
        console.log('模块加载完毕')
    })
</script>

qiankun ​

qiankun是基础的微前端框架，基于single-spa，封装了微前端的实现细节，简化了微前端的配置，让开发者只需要关注微前端的实现，而无需关注微前端的实现细节。

工作原理 ​

  1. 父应用的结构：有一个 <div id='container'></div> 作为子应用的容器。
  2. qiankun 的处理： 当 qiankun 加载子应用时，它会进行以下操作：
    • 创建一个 shadow DOM 或者使用一个简单的 div 作为子应用的沙箱容器。
    • 在这个沙箱容器内，qiankun 会动态创建一个 <div id="root"></div> 元素。
    • 这个新创建的 div 会被插入到父应用中指定的容器（即 <div id='container'></div>）内。
  3. 子应用的渲染： 当子应用的 mount 函数被调用时，qiankun 会传递一个 props 对象，其中包含 container 属性。这个 container 就是 qiankun 为子应用创建的沙箱容器。
  4. 当子应用执行 container.querySelector('#root') 时，它实际上是在 qiankun 创建的沙箱容器内查找 id="root" 的元素，而不是在整个文档中查找。

js沙箱 ​

js沙箱 防止应用加载的时候 对window造成污染

快照实现js沙箱 先保留a应用的属性 失活的时候 把修改的属性存起来 激活的时候还原 缺点：window快照 浪费内存

js
    class SnapshotSandbox {
      constructor() {
        this.modifyPropsMap = {}
      }
      active() {
        this.windowSnapShot = {}
        // 保存当前window的属性
        Object.keys(window).forEach(key => {
          this.windowSnapShot[key] = window[key]
        })
        Object.keys(this.modifyPropsMap).forEach(key => {
          window[key] = this.modifyPropsMap[key]
        })
      }
      inactive() {
        this.modifyPropsMap = {}
        Object.keys(window).forEach(key => {
          if (window[key] !== this.windowSnapShot[key]) {
            this.modifyPropsMap[key] = window[key]
            window[key] = this.windowSnapShot[key]
          }
        })
      }
    }

    let sandbox = new SnapshotSandbox()
    sandbox.active()
    window.a = 100
    sandbox.inactive()
    console.log(window.a) // undefined
    sandbox.active()
    console.log(window.a) // 100

proxy实现js沙箱

js
//proxy 实现沙箱  
    class LegacySandbox {
      constructor() {
        //1.修改的内容 2.新增的内容  3.不管修改还是新增的
        this.modifyPropsMap = new Map()
        this.addedPropsMap = new Map()
        this.currentPropsMap = new Map()
        //访问的时候通过window，设置的时候经过代理保存一遍
        const fakeWindow = Object.create(null)
        const proxy = new Proxy(fakeWindow, {
          get: (target, key, receiver) => {
            return window[key]
          },
          set: (target, key, value) => {
            if (!window.hasOwnProperty(key)) {
              this.addedPropsMap.set(key, value) //新增的属性
            } else if (!this.modifyPropsMap.has(key)) {
              this.modifyPropsMap.set(key, window[key]) //保存旧的内容
            }
            this.currentPropsMap.set(key, value)
            window[key] = value  //修改成最新的内容
          }
        })
        this.proxy = proxy
      }
      setWindowProps(key, value) {
        if (value === undefined) {
          delete window[key]
        } else {
          window[key] = value  //覆盖修改前的
        }
      }

      active() {
        this.currentPropsMap.forEach((value, key) => {
          this.setWindowProps(key, value)
        })
      }

      inactive() {
        this.modifyPropsMap.forEach((value, key) => {
          this.setWindowProps(key, value)
        })
        this.addedPropsMap.forEach((value, key) => {
          this.setWindowProps(key, undefined)
        })
      }
    }


    let legacySandbox = new LegacySandbox()
    legacySandbox.active()
    legacySandbox.proxy.b = 300
    legacySandbox.inactive()
    console.log(window.b) // undefined 
    legacySandbox.active()
    console.log(window.b) // 300

基于proxy实现的沙箱可以节约内存 snapshot和基于proxy实现的sandbox只能单例的情况下使用，同时加载多个应用就会有混乱的风险

多个实例来实现沙箱

js
    class ProxySandbox {
      constructor() {
        this.running = false
        const fakeWindow = Object.create(null)
        this.proxy = new Proxy(fakeWindow, {
          get: (target, key) => {
            return key in target ? target[key] : window[key]
          },
          set: (target, key, value) => {
            if (this.running) {
              target[key] = value
            }
            return true
          }
        })
      }
      active() {
        if (!this.running) this.running = true
      }

      inactive() {
        this.running = false
      }
    }

    let sandbox1 = new ProxySandbox()
    let sandbox2 = new ProxySandbox()
    sandbox1.active()
    sandbox2.active()
    sandbox1.proxy.a = 100
    sandbox2.proxy.a = 200
    console.log(window.a, sandbox1.proxy.a, sandbox2.proxy.a) // undefined 100 200
    sandbox1.inactive()
    sandbox2.inactive()
    sandbox1.proxy.a = 300
    sandbox2.proxy.a = 400
    console.log(window.a, sandbox1.proxy.a, sandbox2.proxy.a) // undefined 100 200
