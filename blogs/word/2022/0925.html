背景 ​

在移动端开发中，设计师经常会要求实现 0.5px 的细边框效果。然而直接使用 border: 0.5px solid #000 会存在兼容性问题，本文将详细介绍原因和解决方案。

为什么会有这个问题？ ​

设备像素比(DPR)的影响 ​

  • PC端：1个CSS像素 = 1个物理像素
  • 移动端：1个CSS像素 = n个物理像素（n为设备像素比）

例如：

  • iPhone 6 物理像素：750 × 1334
  • iPhone 6 逻辑像素：375 × 667
  • 设备像素比(DPR)：2

这意味着在 iPhone 6 上，CSS中写的 0.5px 实际上会被渲染成 1px，因为最小显示单位是 1个逻辑像素。

解决方案 ​

1. 单边框实现方案 ​

方案一：border-image + linear-gradient ​

css
.border-image {
    border-bottom: 1px solid transparent;
    border-image: linear-gradient(to bottom, transparent 50%, #000 50%) 0 0 100% 0;
}

方案二：伪元素 + background-image ​

css
.bg-image::before {
    content: "";
    position: absolute;
    left: 0;
    bottom: 0;
    width: 100%;
    height: 1px;
    background-image: linear-gradient(to bottom, transparent 50%, #000 50%);
}

方案三：伪元素 + transform scale（推荐） ​

css
.scale::after {
    content: "";
    position: absolute;
    left: 0;
    bottom: 0;
    width: 100%;
    height: 1px;
    background: #000;
    transform: scaleY(0.5);
}

2. 多边框实现方案 ​

css
.border-all::before {
    content: "";
    position: absolute;
    left: 0;
    top: 0;
    width: 200%;
    height: 200%;
    border: 1px solid #000;
    transform: scale(0.5);
    transform-origin: 0 0;
}

3. 结合 JS 判断设备像素比 ​

javascript
if (window.devicePixelRatio && devicePixelRatio >= 2) {
    document.querySelector('.target').classList.add('scale-1px');
}

最佳实践建议 ​

  1. 对于单边框，优先使用 transform scale 方案，兼容性好且实现简单
  2. 对于多边框场景，使用 transform scale 整体缩放方案
  3. 建议结合 JS 判断设备像素比，仅在高清屏下应用这些方案

浏览器兼容性 ​

  • transform 和 transform-origin 需要考虑添加浏览器前缀
  • border-image 方案在部分低版本浏览器可能不支持
